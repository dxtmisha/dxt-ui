## Интеграция компонента

Компонент обеспечивает гибкую интеграцию с табличными структурами данных, координируя отображение ячеек на основе переданного списка столбцов и объекта строки.

### Конфигурация полей columns и item

Связывание данных и структуры осуществляется через два ключевых свойства:

- `columns` — массив строк, определяющий последовательность и идентификаторы активных столбцов (например, `['id', 'name', 'role']`).
- `item` — объект данных текущей строки, содержащий значения для отображения, где ключи соответствуют идентификаторам из `columns`.

Для каждого идентификатора столбца из массива `columns` компонент автоматически извлекает соответствующее значение из объекта `item` (вычисляемое как `item[index]`) и производит рендеринг ячейки.

### Работа с избыточными данными

Объект `item` может содержать любые дополнительные или служебные поля (например, технические идентификаторы, связанные объекты, сырые ответы от API). Компонент отфильтрует структуру и выведет исключительно те свойства, ключи которых явно объявлены в массиве `columns`.

Это избавляет от необходимости осуществлять предварительную трансформацию (reshaping) или фильтрацию исходного объекта данных перед его передачей в компонент.

      />
