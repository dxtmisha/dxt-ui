## Description

A utility component for expanding content beyond the horizontal padding of its parent container using negative margins.

Bleed provides an elegant solution for creating "bleed" elements (such as full-width images or banners) inside width-constrained containers. The component automatically calculates the necessary negative margins based on the provided value, allowing content to visually break through parent padding boundaries without requiring layout restructuring.

**Key Features:**

- Automatic application of negative horizontal margins
- Support for custom HTML tag configuration
- Preservation of the document's vertical flow
- Integration with the design token system
- Responsiveness through dynamic margin management

**Typical Usage Scenarios:**

- Full-width images in articles and long-reads
- Banners and breakout sections in text blocks
- Horizontal dividers extending to the edges
- Integration of third-party widgets requiring more space
- Decorative elements breaking out of the grid

> Bleed simplifies complex layout management by allowing content to dynamically expand beyond parent container boundaries while maintaining layout integrity and semantic structure.
