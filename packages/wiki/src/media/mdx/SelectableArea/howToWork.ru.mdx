## Интеграция компонента

Компонент `SelectableArea` построен по принципу *renderless*-архитектуры. Он инкапсулирует сложную логику выделения (вычисление координат, работу с лассо, обработку модификаторов клавиатуры), но полностью делегирует визуальный рендеринг вашему шаблону через свойства слота по умолчанию.

Для корректной работы механизма выделения необходимо связать передаваемые свойства с целевыми DOM-узлами.

### Конфигурация элементов

Каждый интерактивный элемент внутри области должен содержать следующие привязки:

- `className` — базовый идентификатор. Добавляется в атрибут `class`. Необходим для регистрации элемента во внутренней системе координат компонента.
- `classClick` — идентификатор интерактивной зоны. Добавляется в атрибут `class`. Указывает компоненту, какая именно область элемента должна реагировать на события мыши.
- `data-value` — уникальный ключ элемента (строка или число). Передается как стандартный HTML-атрибут. Именно это значение добавляется в массив `selected` при выделении.
- `onClick` — внутренний обработчик событий. Передается в директиву `@click` и отвечает за логику выделения одиночным кликом, а также множественного выбора с использованием клавиш Shift и Cmd/Ctrl.

### Управление визуальным состоянием

В целях максимальной производительности (особенно при обработке сотен элементов с помощью лассо) `SelectableArea` **самостоятельно** манипулирует DOM-деревом, напрямую добавляя и удаляя классы активного состояния. Это позволяет избежать дорогостоящих циклов рендеринга виртуального DOM во Vue.

#### Кастомная логика стилизации (`classSelection`)
Свойство `classSelection`, предоставляемое слотом, **не является обязательным** для базовой работы компонента. Оно предназначено исключительно для продвинутых случаев, когда архитектура вашего приложения требует переопределения стандартного поведения. Вы можете использовать `classSelection` для применения готовых дизайн-токенов к элементам, чье активное состояние контролируется вашей собственной внешней бизнес-логикой.

<Source
    code={`
<template #default="{ className, classClick, classSelection, onClick }">
  <div class="my-grid-container">
    <div
      v-for="item in items"
      :key="item.id"
      :class="[
        className,
        classClick,
        /* Используйте classSelection только если внедряете собственную логику отображения состояния */
        isCustomActive(item) ? classSelection : ''
      ]"
      :data-value="item.id"
      @click="onClick"
    >
      {{ item.name }}
    </div>
  </div>
</template>
  `}
