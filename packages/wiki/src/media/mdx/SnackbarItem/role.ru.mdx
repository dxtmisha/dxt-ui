## ARIA роль

Свойство `role` позволяет переопределить стандартную ARIA-роль для элемента уведомления. По умолчанию используется роль `status`.

**Когда использовать:**
 - `status` (по умолчанию) — используйте для сообщений, которые содержат вспомогательную информацию, не требующую немедленного действия. Ассистивные технологии (экранные дикторы) озвучат такое сообщение, когда пользователь будет свободен.
 - `alert` — используйте для критических ошибок или предупреждений, требующих немедленного внимания. Экранные дикторы прервут текущую задачу, чтобы озвучить это уведомление.
 - `log` — используйте, если уведомления представляют собой последовательный лог событий.

<Source
    code={`
<template>
  <!-- Стандартное уведомление -->
  <SnackbarItem label="Загрузка завершена" role="status" />

  <!-- Критическая ошибка -->
  <SnackbarItem label="Ошибка соединения!" role="alert" />
</template>
  `}
    language="html"
/>
