## Описание

Компонент для динамического переключения контента на основе контекста области (area).

Area выступает в роли умного переключателя слотов. Он получает текущее значение области из контекста внедрения или использует значение по умолчанию. Затем компонент отображает тот слот, имя которого совпадает с текущим значением области. Если подходящий слот не найден, отображается слот по умолчанию.

**Основные возможности:**

- Автоматический выбор слота на основе внедренного значения области
- Возможность указания области по умолчанию через свойство `areaDefault`
- Поддержка вложенных областей и наследования контекста
- Плавная интеграция в сложные макеты (например, Window)

**Типичные сценарии использования:**

- Создание динамических шапок и подвалов
- Адаптация контента в зависимости от места размещения компонента
- Организация сложных макетов с динамическими зонами контента
- Реализация слотов расширения в базовых компонентах

<Source
    code={`
<template>
  <Area area-default="header">
    <template #header>
      Контент для шапки
    </template>
    <template #footer>
      Контент для подвала
    </template>
    <template #default>
      Контент по умолчанию
    </template>
  </Area>
</template>
  `}
    language="html"
/>

> Area является важным инструментом для создания гибких и расширяемых интерфейсов, позволяя компонентам реагировать на их окружение.
