Component for creating interactive radio buttons designed to select a single value from a group of proposed options.

Radio manages the selection state of elements, supports validation, integration with forms, customizable icons, and transition animations. The component automatically handles state changes, focus management, and event generation during user interaction. Unlike a checkbox, a radio button is typically used in a group where only one item can be selected.

**Key features:**

- Selection of a single value from a group (exclusive selection)
- Built-in validation with custom error messages
- Support for labels and descriptions
- Customizable icons for different states
- Integration with forms via name and value
- Disabled, loading, and skeleton states
- Ripple effect on interaction
- Adaptive layout and styling

**Common use cases:**

- Selection of gender or marital status
- Setting delivery or payment methods
- Switching between operating modes
- Surveys with a single answer option
- Settings forms where parameters are mutually exclusive
