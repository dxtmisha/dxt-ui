### `trailing`

Слот для размещения пользовательского контента в конце интерактивной области поля (после основного содержимого ввода, но до внутренних управляющих иконок).

- Расположен логически в конце (справа в LTR, слева в RTL)
- Подходит для иконок действий (очистка, открыть, переключить), счётчиков, небольших статусных бейджей
- Получает тот же объект props (`{ id, className, classHidden }`), что и слот `default`
- Должен быть компактным — избегайте крупных элементов, ломающих компоновку
- Не заменяет `suffix` — используйте `suffix` для семантического завершающего текста, `trailing` — для функционального / динамического UI
