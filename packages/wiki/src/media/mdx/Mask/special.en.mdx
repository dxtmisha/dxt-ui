## Special characters

The `special` property defines special characters to denote user input positions in the mask and configure their extended behavior.

**Possible values:**

- `string` — single special character (default `'*'`)
- `string[]` — array of special characters for different group types
- `MaskSpecialList` — object with extended configuration for each character

Special characters in the mask template denote positions where users can enter data, unlike literal characters that are inserted automatically.

When using a simple string (e.g., `special="*"`), all input positions in the mask are restricted by the same validation rules.

An array of characters (e.g., `special="['*', 'A', '#']"`) allows creating different group types with individual constraints: one group for digits, another for letters, a third for special characters.

Object notation `MaskSpecialList` (see [detailed description below](#maskspeciallist)) provides maximum flexibility. Each special character is configured separately with the ability to set default values, rubber groups with variable length, auto-transition characters, and custom validation.

For built-in types (`currency`, `number`, `date`, `time`), the system automatically generates special characters considering regional settings, ignoring the user-defined `special` value.

<Source
    code={`
<!-- Simple special character -->
<Mask mask="***-***" special="*" />

<!-- Multiple group types -->
<Mask mask="AA*-###" :special="['A', '*', '#']" />

<!-- Extended configuration -->
<Mask
  mask="***-***"
  :special="{
    '*': {
      pattern: '\\d',
      view: '_',
      rubber: true,
      minLength: 2,
      maxLength: 8,
      transitionChar: ' '
    }
  }"
/>

<!-- Groups with default values -->
<Mask
  mask="DD/MM/YYYY"
  :special="{
    'D': { defaultValue: '01', pattern: '\\d' },
    'M': { defaultValue: '01', pattern: '\\d' },
    'Y': { defaultValue: '2024', pattern: '\\d' }
  }"
/>

## MaskSpecialList

`MaskSpecialList` is an extended configuration object that allows for fine-tuned control over the behavior of each special character in the mask. Each key in the object corresponds to one of the characters defined in the `special` property (or the default `*` character).

### MaskSpecialItem Properties

- `defaultValue?: string` — **initial value**. Automatically populates the group on initialization. The user can modify it if the `match` rules allow.
- `rubber?: boolean` — **rubber mode**. Enables dynamic length for the group. In the mask template, such a group is typically represented by one or more identical characters.
- `transitionChar?: string | string[]` — **transition character**. A specific character (e.g., space or dot) that triggers an immediate focus shift to the next part of the mask upon entry. Only for `rubber: true`.
- `minLength?: number` — **minimum length**. Used to validate whether the group has been sufficiently filled.
- `maxLength?: number` — **maximum length**. The limit for expansion in "rubber" groups.
- `match?: RegExp | string` — **input filter**. A regular expression that checks *each individual* character as it is typed. Default: `[0-9]`.
- `pattern?: string | Function` — **group validator**. A rule for validating the entire accumulated string in the group. Can be a string (regex) or a function receiving mask data.
- `view?: string` — **visual placeholder**. The character the user sees in empty slots of this group. Default: `_`.

### How to Work with MaskSpecialList

For effective use of `MaskSpecialList`, follow these simple rules:

1.  **Character Mapping**: Every key in the `MaskSpecialList` object must be a character you used in your `mask` template. For example, if the mask is `AAA-***`, you can configure group `A` for letters and `*` for digits.
2.  **Filter Configuration (`match`)**: Use `match` to restrict the types of characters a user can physically type. This stops letters from being entered where numbers are expected at the earliest possible stage.
3.  **Use Rubber Groups**: If a field's length is unknown in advance (like a phone extension), set `rubber: true`. This allows the mask to expand as the user types.
4.  **Manage Transitions**: For a better user experience, define a `transitionChar` (like a space or dot). This allows users to manually finish the current group and jump to the next one without filling the entire length.
5.  **Default Values**: If part of the data is frequently repeated (like the current year), use `defaultValue`. This reduces the number of actions required from the user.

### Example

<Source
    code={`
<script setup>
const value = ref('')
</script>

<template>
  <!-- Example of a rubber mask for an IP address -->
  <Mask
    v-model="value"
    mask="*.#.#.#"
    :special="{
      '*': { rubber: true, maxLength: 3, transitionChar: '.', pattern: '\\d{1,3}' },
      '#': { rubber: true, maxLength: 3, transitionChar: '.', pattern: '\\d{1,3}' }
    }"
  />

  <!-- Example of a mask with different group types and default values -->
  <Mask
    mask="AAA-****"
    :special="{
      'A': { match: /[A-Z]/, view: 'A' },
      '*': { defaultValue: '0000', match: /[0-9]/, view: '0' }
    }"
  />
</template>
  `}
    language="html"
/>
