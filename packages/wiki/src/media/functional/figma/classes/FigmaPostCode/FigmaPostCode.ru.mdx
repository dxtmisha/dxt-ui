# Класс FigmaPostCode

Утилитарный класс для управления уникальным кодом верификации, используемым для обеспечения целостности сообщений между плагином Figma (backend) и его UI (frontend). Это гарантирует, что сообщения, полученные одной стороной, действительно были отправлены другой стороной той же сессии плагина.

## Ключевые особенности

- **Безопасность** — Предотвращает вмешательство сообщений от других плагинов.
- **Блокировка сессии** — Код может быть установлен только один раз за сессию, что обеспечивает согласованность.
- **Автогенерация** — По умолчанию генерируется случайный код (например, `figma-123456`), если он не установлен вручную.

## Статические методы

### `is`

Проверяет, совпадает ли предоставленный код с текущим кодом сообщения.

**Параметры:**
- `code: string` — Строка проверочного кода для сверки с внутренним кодом сессии.

**Возвращает:** `boolean` — `true`, если код совпадает.

### `get`

Возвращает текущий код сообщения, используемый для верификации в текущей сессии.

**Возвращает:** `string`

### `set`

Устанавливает новый код сообщения. Этот метод может быть успешно выполнен только один раз; как только код установлен (или после первого обращения к `get`), сессия блокируется, и код больше не может быть изменен.

**Параметры:**
- `code: string` — Новое значение кода для использования в текущей сессии.

**Возвращает:** `void`
