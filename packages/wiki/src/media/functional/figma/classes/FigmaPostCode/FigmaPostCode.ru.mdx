# Класс FigmaPostCode

Утилитарный класс для управления уникальным кодом верификации, используемым для обеспечения целостности сообщений между плагином Figma (backend) и его UI (frontend). Это гарантирует, что сообщения, полученные одной стороной, действительно были отправлены другой стороной той же сессии плагина.

## Ключевые особенности

- **Безопасность** — Предотвращает вмешательство сообщений от других плагинов.
- **Блокировка сессии** — Код может быть установлен только один раз за сессию, что обеспечивает согласованность.
- **Автогенерация** — По умолчанию генерируется случайный код (например, `figma-123456`), если он не установлен вручную.

## Статические методы

- `is(code: string): boolean` — Проверяет, совпадает ли предоставленный код с текущим кодом сообщения.
- `get(): string` — Возвращает текущий код сообщения.
- `set(code: string): void` — Устанавливает новый код сообщения. Можно вызвать только один раз.

## Рекомендации

Для обеспечения стабильной и уникальной верификации рекомендуется использовать `id` из вашего файла `manifest.json`.

```typescript
import manifest from '../manifest.json'
import { FigmaPostCode } from '@dxtmisha/figma'

// Установка кода верификации с использованием ID из манифеста
FigmaPostCode.set(manifest.id)
```
