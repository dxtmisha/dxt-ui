# FigmaPostAbstract Class

Base abstract class for managing messaging between the Figma plugin (backend) and the UI (frontend). It provides a structured foundation for bidirectional communication with built-in verification and multiple listener management.

## Key Features

- **Bidirectional Support** ‚Äî Foundation for both `FigmaPluginMessenger` and `FigmaUiMessenger`.
- **Message Verification** ‚Äî Uses `FigmaPostCode` to ensure that messages originated from the same plugin instance.
- **Multicast Listeners** ‚Äî Supports multiple callbacks for a single message type.
- **Method Chaining** ‚Äî Methods like `make` and `add` return `this`, allowing for a more concise initialization.

## Public Methods

### `make`

Initializes the message listener. This method ensures that the environment-specific setup (`prepare`) is executed exactly once, regardless of how many times `make` is called.

**Returns:** `this`

```javascript
const messenger = new FigmaPluginMessenger().make()
```

### `add`

Registers a callback function to be executed when a message of a specific type is received.

**Parameters:**
- `type: string` ‚Äî The unique identifier for the message type.
- `callback: UiFigmaMessengerCallback<Message>` ‚Äî The function to call when the message is received. Receives the message data as its first argument.
- `once: boolean = false` ‚Äî If set to `true`, the callback will be automatically removed after being called once.

**Returns:** `this`

```javascript
messenger
  .add('update-selection', (data) => console.log('Selection:', data.count))
  .add('init', (data) => console.log('Init:', data), true)
  .add('close', () => figma.closePlugin())
```

### `remove`

Removes a previously registered callback function for a specific message type.

**Parameters:**
- `type: string` ‚Äî The unique identifier for the message type.
- `callback: UiFigmaMessengerCallback<Message>` ‚Äî The function to be removed from the listeners.

**Returns:** `this`

```javascript
const onUpdate = (data) => console.log('Selection:', data.count)

messenger
  .add('update-selection', onUpdate)
  .remove('update-selection', onUpdate)
```

## Data Structures

### `UiFigmaMessengerData`

The structure of the data object passed through the messaging pipeline:

- `code: string` ‚Äî The verification code from `FigmaPostCode`.
- `type: string` ‚Äî The identifier of the message.
- `message: Message` ‚Äî The actual payload data.
