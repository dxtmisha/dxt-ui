# Система направления текста

Библиотека `@dxtmisha/styles` предоставляет мощный механизм для поддержки двунаправленного текста и раскладок (LTR и RTL) с использованием динамических CSS Custom Properties (CSS-переменных). Вместо дублирования CSS-кода или жесткого прописывания статических свойств, эта система автоматически управляет выравниванием, отступами, позиционированием и трансформациями прямо в браузере.

Классический подход к зеркалированию раскладок требует написания отдельных переопределяющих стилей вручную (например, переопределение margin-left и margin-right). Данная утилита решает эту проблему путем привязки физических направлений к CSS-переменным, которые автоматически обновляются при изменении направления.

## Как это работает

Система направления текста построена на динамическом обмене значений переменных:

1. **Глобальная инициализация**: По умолчанию (в режиме LTR) библиотека задает переменные на уровне `html` (например, `--d-dir: 1` для смещений, выравнивания вроде `--d-dir-start: flex-start` и флаги стороны `--d-dir-left: 1`).
2. **Автоматическое переключение**: Для каждого дочернего элемента физические отступы привязываются к динамическим переменным (например, `--d-dir-margin-left` считывает `var(--sys-margin-left)`).
3. **Обмен при RTL**: При активации RTL (через `html[dir="rtl"]` или `.dir-rtl`) система меняет значения флагов (`--d-dir` становится `-1`, `--d-dir-start` переключается на `flex-end`) и меняет местами переменные свойств на дочерних элементах (например, `--d-dir-margin-left` считывает `var(--sys-margin-right)`).

## Инициализация системы (`initDir`)

Для запуска системы направлений необходимо вызвать миксин `initDir` на глобальном уровне (обычно в файле инициализации глобальных стилей):

```scss
@use '@dxtmisha/styles';

@include styles.initDir;
```

Этот миксин задает переменные LTR для элемента `html` по умолчанию, настраивает активные свойства направлений для всех дочерних элементов (`*`, `*::before`, `*::after`), а также добавляет триггеры, которые автоматически применяют миксин `dirToRtl` при добавлении к документу атрибута `[dir="rtl"]` или класса `.dir-rtl`.

## Функции

- `isDir(): String` — Возвращает множитель направления (`1` для LTR, `-1` для RTL).
- `isDirLeft(): String` — Возвращает `1` для LTR, `0` для RTL.
- `isDirRight(): String` — Возвращает `0` для LTR, `1` для RTL.
- `dirLeft(): String` — Возвращает `left` для LTR, `right` для RTL.
- `dirRight(): String` — Возвращает `right` для LTR, `left` для RTL.
- `dirLeftAuto(): String` — Возвращает `auto` для LTR, `unset` для RTL.
- `dirRightAuto(): String` — Возвращает `unset` для LTR, `auto` для RTL.
- `dirFlexLeft(): String` — Возвращает `flex-start` для LTR, `flex-end` для RTL.
- `dirFlexRight(): String` — Возвращает `flex-end` для LTR, `flex-start` для RTL.
- `dirToLeft($left, $right): String` — Возвращает значение перехода из левого положения в правое.
- `dirToRight($left, $right): String` — Возвращает значение перехода из правого положения в левое.
- `dirOrder($value): String` — Вычисляет значение order на основе направления.

## Миксины

- `initDir()` — Инициализирует систему направлений для поддержки LTR и RTL.
- `dirToRtl()` — Применяет стили и переменные для RTL направления.
