# Работа с сообщениями (Messenger)

Механизм обмена сообщениями является центральным звеном в архитектуре плагинов Figma. Поскольку «сторона кода» (sandbox) и «сторона интерфейса» (UI) работают в изолированных средах, любое взаимодействие между ними требует использования системы `postMessage`.

Библиотека упрощает этот процесс, предоставляя стандартизированные мессенджеры и автоматическую маршрутизацию.

### Зачем это нужно?
Прямое использование `figma.ui.postMessage` и `window.addEventListener` имеет ряд недостатков:
- Отсутствие типизации передаваемых данных.
- Сложность в отслеживании отправителя и получателя при большом количестве типов сообщений.
- Необходимость вручную фильтровать сообщения, предназначенные для конкретной логики.

Система мессенджеров решает эти задачи, инкапсулируя низкоуровневую работу с событиями.

### Как это работает

Архитектура построена на использовании двух зеркальных классов:
1. **`FigmaPluginMessenger`** — используется на стороне кода (backend).
2. **`FigmaUiMessenger`** — используется на стороне интерфейса (frontend).

Оба класса наследуются от `FigmaPostAbstract` и обеспечивают единый API для отправки и получения данных.

#### 1. Определение типов сообщений
Для обеспечения согласованности рекомендуется выносить все типы сообщений в отдельный файл, доступный как для стороны кода, так и для стороны UI вашего плагина (например, в папку `shared/types`).

```typescript
// plugin/shared/types.ts
export const UI_FIGMA_MY_ACTION = 'my-action-type'
```

Это позволит использовать одну и ту же константу в обоих окружениях, избегая опечаток.

#### 2. Отправка и получение на стороне Плагина (Code)
На стороне кода мы подписываемся на события и отправляем результаты обратно.

```typescript
import { FigmaPluginMessenger } from '@dxtmisha/figma-code'
import { UI_FIGMA_MY_ACTION } from '@dxtmisha/figma'

const messenger = FigmaPluginMessenger.getInstance()

// Подписка на запрос из UI
messenger.add(UI_FIGMA_MY_ACTION, (data) => {
  console.log('Получены данные:', data)
  
  // Отправка ответа обратно в UI
  messenger.post(UI_FIGMA_MY_ACTION, { status: 'success' })
})
```

#### 3. Отправка и получение на стороне UI (Vue)
В интерфейсе мы используем тот же подход через `FigmaUiMessenger`.

```typescript
import { FigmaUiMessenger } from '@dxtmisha/figma'
import { UI_FIGMA_MY_ACTION } from '@dxtmisha/figma'

const messenger = FigmaUiMessenger.getInstance()

// Отправка запроса в плагин
messenger.post(UI_FIGMA_MY_ACTION, { payload: 'hello' })

// Подписка на ответ
messenger.add(UI_FIGMA_MY_ACTION, (result) => {
  console.log('Результат выполнения:', result)
})
```

### Резюме
Использование системы мессенджеров позволяет:
- **Централизовать связь**: Вся коммуникация проходит через единые точки входа.
- **Гарантировать доставку**: Каждое сообщение содержит идентификатор `code`, который проверяется системой для исключения конфликтов с другими плагинами.
- **Упростить отладку**: Единый формат логирования и обработки ошибок.
