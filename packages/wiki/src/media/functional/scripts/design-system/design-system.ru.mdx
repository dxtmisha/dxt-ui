# Работа с дизайн системой

Пакет `@dxtmisha/scripts` — это мощный инструмент и "мозг" автоматизации всей дизайн-системы **DXT UI**. Это набор утилит и команд CLI, который берет на себя рутинную работу по созданию, связыванию, тестированию и документированию компонентов.

Главная цель этого пакета — дать разработчикам и бизнесу единый, автоматизированный процесс развития UI-кита, где человеческий фактор и ошибки при написании шаблонного кода (boilerplate) сведены к нулю. 

Особая ценность пакета заключается в том, что он выступает **мостом между абстрактной бизнес-логикой и визуальным дизайном**.

## Разделение логики и дизайна

Архитектура DXT UI построена на строгом разделении:
- **Абстрактная логика (Конструктор)** — изолированные классы, которые управляют ARIA-атрибутами, событиями, фокусом и состояниями, но ничего не знают о цветах, размерах или CSS-классах.
- **Визуальный дизайн** — конкретное воплощение (токены, отступы, палитры).

Пакет `scripts` берет на себя задачу автоматически и безошибочно "сшить" эти два мира.

## Как это работает под капотом:

1. **`properties.json` как единый источник правды (Single Source of Truth):**
   Разработчик или дизайнер описывает только визуальные свойства компонента (токены, цвета, размеры, состояния `:hover`/`:active`, модификаторы) в одном конфигурационном файле `properties.json`.
   
2. **Парсинг и кодогенерация (Scaffolding):**
   Пакет `scripts` читает этот конфигурационный файл и автоматически генерирует всю связующую инфраструктуру:
   - `props.ts` — TypeScript-интерфейсы, которые расширяют базовые свойства из Конструктора, добавляя в них точные типы для дизайна (например, `size: 'sm' | 'md'`) и значения по умолчанию.
   - `styleToken.scss` — готовые SCSS-миксины и CSS-переменные (дизайн-токены).
   - `DesignComponent.vue` — Vue-шаблон, в котором происходит магия: скрипт генерирует вычисляемые BEM-классы на основе свойств и прокидывает их в абстрактный класс `Design` из Конструктора (внедрение зависимостей).
   
3. **Оркестрация типов (`dxt-constructor`):**
   Добавление нового свойства в `properties.json` автоматически обновляет глобальные интерфейсы проекта. IDE сразу начинает подсказывать новые свойства при использовании компонента (например, через `web-types.json`).

4. **Генерация документации с помощью ИИ (AI-Docs):**
   Интеграция с искусственным интеллектом (Google Gemini / Anthropic Claude) позволяет пакету автоматически генерировать справочные материалы, описания пропсов и примеры кода прямо на основе `properties.json` и типов TypeScript.

## Что это дает бизнесу и команде разработчиков?

- **Высокая скорость разработки:** Фокус смещается с написания бойлерплейта на создание реального продукта. Создание нового компонента или модификатора занимает секунды.
- **Гарантированная консистентность:** Исключаются "забытые" экспорты, опечатки в именах CSS-классов или расхождения между TypeScript и SCSS. Логика (например, доступность ARIA) всегда работает идеально, так как наследуется от Конструктора, а визуальная часть 100% соответствует токенам.
- **Простота поддержки и масштабирования:** Чтобы изменить логику генерации CSS-классов или структуру папок во всей библиотеке, достаточно внести правки в логику пакета `scripts`, после чего вся библиотека UI пересобирается одной командой.
- **Интеграция с дизайном (Figma):** Встроенные клиенты Figma API позволяют извлекать структуры фреймов, дизайн-токены и стили напрямую из макетов, автоматически превращая их в код.

## Основные команды (CLI)

- **`npm run ui [ComponentName]` / `dxt-ui`**
  Главный оркестратор. Сканирует файлы `properties.json`, генерирует шаблоны, "сшивает" их с классами Конструктора, обновляет пропсы и стили. 
- **`npm run constructor` / `dxt-constructor`**
  Пересобирает глобальные классы и типы конструктора логики, гарантируя актуальность всех типов свойств.
- **`dxt-component-wiki` / `dxt-wiki-storm`**
  Запускает генерацию ИИ-справок по компонентам для Storybook/MDX и форматирует данные для автозаполнения в IDE (WebStorm).
- **`npm run library` / `dxt-library`**
  Сканирует дерево директорий и автоматически собирает файлы экспорта `index.ts` во всех пакетах рабочего пространства.
- **`dxt-screenshot`**
  Среда визуального регрессионного тестирования с использованием Puppeteer.

## Глубокая интеграция стилей и дизайн-токенов

Модуль парсинга дизайн-токенов (классы `Properties*`) — это сердце кодогенератора. Скрипт распознает множество типов свойств из JSON:
- **`type-var` / `flag-var`**: Свойства, компилируемые напрямую в CSS-переменные (динамическое управление отступами, скруглениями).
- **`type-state`**: Динамические визуальные состояния компонентов.
- **`flag-theme` / Палитры**: Управление цветами и поддержка темной/светлой темы без дублирования CSS.
- **`flag-subclass` / `flag-replace`**: Умное управление CSS-классами по BEM, добавляющее модификаторы только при активных флагах.

Все эти связи скрипт автоматически переводит в оптимизированный SCSS-код и строгий TypeScript, образуя безупречный мост между визуальным дизайном и логическим ядром компонентов.
