# dxt-types

Команда `dxt-types` собирает, очищает и сжимает файлы деклараций TypeScript (`.d.ts`) в оптимизированный для ИИ справочник типов. Она отфильтровывает внутренние детали реализации и использует ИИ для генерации инструкций разработчика.

## Ключевые возможности

- **Обработка деклараций**: Рекурсивно собирает файлы `.d.ts` из папки вывода сборки (например, `dist/`).
- **Фильтрация инкапсуляции**: Автоматически удаляет приватные, защищённые (protected) и неэкспортируемые члены, оставляя только публичный интерфейс (API).
- **Оптимизация для ИИ**: Использует ИИ-модель для очистки комментариев и создания компактного, экономичного по токенам файла `ai-types.md`.
- **Создание технического описания**: Запрашивает у ИИ структурированное руководство по возможностям библиотеки и сохраняет его в `ai-developer.md`.

## Как это работает?

1. **Сканирование деклараций**: Скрипт обходит целевую папку сборки в поисках файлов `.d.ts`, игнорируя декларации Vue-компонентов и вики.
2. **Объединение кода**: Соединяет все подходящие файлы в единый текстовый поток в памяти.
3. **Очистка с помощью ИИ (API)**: Отправляет текст ИИ для удаления комментариев, импортов, скрытых методов и сжатия структуры.
4. **Сохранение типов**: Записывает очищенные типы в `ai-types.md` вместе с правилами использования экспортов.
5. **Создание описания**: Заставляет ИИ написать техническое резюме библиотеки и сохраняет его в файл `ai-developer.md`.

## Использование

Запустите сборщик типов через `npx`:
```bash
npx dxt-types [build_dir]
```
- `build_dir` (Необязательно): Директория для сканирования. По умолчанию `dist`.
