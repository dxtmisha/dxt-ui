# dxt-screenshot

Команда `dxt-screenshot` автоматизирует визуальное сохранение макетов интерфейса. Она запускает локальный сервер разработки проекта, открывает headless-браузер, делает снимок экрана страницы в высоком разрешении и извлекает её статический HTML и CSS.

## Ключевые возможности

- **Автоматический жизненный цикл сервера**: Самостоятельно запускает dev-сервер (`npm run dev`) в фоновом процессе и корректно останавливает его после захвата экрана.
- **Снимок в headless-браузере**: Использует Puppeteer через обёртку `BrowserItem` для рендеринга страниц с большой высотой (по умолчанию 12000px) и сохранения снимка в формате `.webp`.
- **Экспорт ассетов**: Сохраняет HTML-структуру в файл `screenshot-code.html` и сгенерированный CSS в `screenshot-styles.css` для анализа стилей.

## Как это работает?

1. **Запуск dev-сервера**: Скрипт запускает процесс `npm run dev` в фоне.
2. **Получение URL**: Сканирует вывод терминала для определения локального адреса хоста (например, `http://localhost:5173`).
3. **Захват страницы**: После обнаружения адреса запускает Puppeteer и переходит по полученному URL.
4. **Сохранение результатов**:
   - Делает скриншот всей страницы и сохраняет изображение.
   - Сохраняет тело DOM в `[path]-code.html`.
   - Сохраняет таблицы стилей в `[path]-styles.css`.
5. **Остановка**: Принудительно завершает фоновый процесс dev-сервера.

## Использование

Запустите инструмент создания скриншота через `npx`:
```bash
npx dxt-screenshot
```
Результаты захвата будут сохранены в папке `./ai-screenshot/`.
