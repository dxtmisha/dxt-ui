# dxt-ui

Команда `dxt-ui` является центральным оркестратором для сборки и инициализации библиотеки компонентов пользовательского интерфейса (UI) дизайн-системы. Она координирует генерацию стилей, создание документации, инициализацию структуры компонентов, экспорт пакетов и типизацию API.

## Key Features

- **Создание и обновление компонентов**: Инициализирует новые компоненты или пакетно обновляет структуру существующих компонентов и SCSS-файлов.
- **Управление экспортом пакета**: Автоматически настраивает `package.json` библиотеки с правильным маппингом экспорта для стилей SASS, плагинов, медиа-ресурсов и TypeScript-типов.
- **Единый конвейер сборки**: Последовательно запускает сборку стилей (`Styles`), генерацию документации (`DesignWiki`), экспорт типов и списков медиа.

## Как это работает?

1. **Очистка кэша**: Скрипт сбрасывает локальные кэши сборки свойств.
2. **Сборка SASS и Вики**: Выполняет сборку глобальных SCSS-переменных и компилирует базовую документацию компонентов.
3. **Инициализация компонентов**:
   - Если передано имя компонента `name`, создается/обновляется структура этого конкретного компонента (HTML-шаблоны, скрипты, SCSS-файлы).
   - Если имя не передано, пакетно обновляются все зарегистрированные компоненты.
4. **Обновление package.json**: Записывает пути экспорта для CSS, SASS, Vue-плагинов и типов в файл `package.json`.
5. **Сборка ассетов**: Запускает генерацию вспомогательных файлов медиа, плагинов и списков типов.

## Использование

Запустите оркестратор через `npx`:
```bash
npx dxt-ui [component-name]
```
- `component-name` (Необязательно): Имя конкретного компонента для инициализации или обновления. Если опущено, обновляются все компоненты.
