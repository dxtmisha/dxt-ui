# Формирование документации для ИИ

Современная разработка всё чаще опирается на инструменты искусственного интеллекта (GitHub Copilot, Cursor, кастомные LLM-агенты). Однако сложные архитектуры, глубокая вложенность файлов и избыточные комментарии часто сбивают ИИ с толку или переполняют его контекстное окно.

Пакет `@dxtmisha/scripts` решает эту проблему, автоматически генерируя оптимизированные инструкции, типы и метаданные, специально сжатые и отформатированные для восприятия нейросетями.

## Оптимизация типов (DesignTypes)

Класс `DesignTypes` — это интеллектуальный "сборщик" типов проекта для передачи их в контекст ИИ.

**Как это работает:**
1. **Сканирование сборки:** Скрипт анализирует директорию `dist/`, находя все файлы деклараций (`.d.ts`), за исключением служебных файлов (wiki, базовых типов).
2. **Объединение:** Найденные файлы склеиваются в единый текстовый массив.
3. **ИИ-компрессия (AI Edit):** Скрипт отправляет полученный код внутреннему ИИ (через `useAi`) со строгим промптом:
   - Удалить все комментарии на русском языке (для экономии токенов и универсальности).
   - Сократить английские комментарии до минимума без потери смысла.
   - Удалить все `private` и `protected` методы, оставив только публичный API.
   - Удалить импорты и огромные перечисления (Enums), не дающие практической пользы.
   - Сохранить все важные JSDoc-теги (`@example`, `@remarks`).
4. **Результат:** Генерируется файл `ai-types.md`, содержащий предельно сжатую, но исчерпывающую библиотеку типов, которую ИИ-ассистенты могут прочитать за долю секунды, чтобы понимать архитектуру пакета и безошибочно писать код.

## Генерация описаний (AI Description)

Помимо типов, скрипт анализирует код и автоматически генерирует `ai-description.md`. Это высокотехничное саммари проекта, которое:
- Объясняет другому ИИ-ассистенту главную цель библиотеки.
- Перечисляет основные классы, хуки и утилиты.
- Указывает строгие триггеры — в каких именно случаях ИИ обязан прочитать `ai-types.md` перед написанием кода.

## Генерация ИИ-инструкций (AiDoc)

В системе заложен модуль `AiDoc`, который рекурсивно обходит директории компонентов и с помощью ИИ генерирует или обновляет markdown-инструкции для разработчиков (например, `ai-developer.md`). 

Класс `AiDocType` анализирует исходный код конкретных директорий и формирует правила поведения: как правильно расширять конструктор, какие классы (`Include`) использовать, какие паттерны запрещены. 

## Основные команды
- **`dxt-types`** — запуск генерации и компрессии TypeScript-типов (`ai-types.md`).
- **`dxt-ai-doc`** — запуск генерации ИИ-документации по директориям проекта.
