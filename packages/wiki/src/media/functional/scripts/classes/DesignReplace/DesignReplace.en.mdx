# DesignReplace
The `DesignReplace` class is a core utility for transforming templates into functional code. It performs a wide range of hierarchical string replacements, substituting placeholders for component names, paths, types, default values, and CSS classes based on the component's structural data.

### Primary Methods
- `get()`: Returns the final processed template string after all replacements.
- `replaceName()`, `replacePath()`: Substitute component name and file path placeholders.
- `replaceType()`, `replaceDefault()`, `replaceProps()`: Inject TypeScript types, default values, and property definitions into the template.
- `replaceClasses()`: Populates the template with available CSS classes from the structure.
- `replaceConstructorClasses()`: Specifically handles class replacements for constructor-based templates.
- `replaceMark()`: A generic method for replacing custom markers with data arrays.

### Utility Methods
- `getRoot()`: Returns the module import path.
- `getNameFile()`: Standardizes file names based on the component.
- `replaceOnce()`: Performs a single replacement for a specific marker.
- `replacePropsValues()`, `replaceClassesValues()`, `replaceStylesValues()`: Transform specific property types into optimized lists or arrays.
- `isBoolean()`, `isNoMark()`: Helper methods for type checking and marker validation.
- `getDefault()`: Retrieves default values for specific structural items.
