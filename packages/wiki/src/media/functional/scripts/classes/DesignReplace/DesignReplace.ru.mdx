# DesignReplace
Класс `DesignReplace` является основным инструментом для преобразования шаблонов в функциональный код. Он выполняет широкий спектр иерархических замен строк, подставляя заполнители для имен компонентов, путей, типов, значений по умолчанию и CSS-классов на основе структурных данных компонента.

### Основные методы
- `get()`: Возвращает окончательную обработанную строку шаблона после всех замен.
- `replaceName()`, `replacePath()`: Заменяют заполнители для имени компонента и пути к файлу.
- `replaceType()`, `replaceDefault()`, `replaceProps()`: Внедряют типы TypeScript, значения по умолчанию и определения свойств в шаблон.
- `replaceClasses()`: Заполняет шаблон доступными CSS-классами из структуры.
- `replaceConstructorClasses()`: Специально обрабатывает замены классов для шаблонов, основанных на конструкторах.
- `replaceMark()`: Универсальный метод для замены пользовательских меток массивами данных.

### Вспомогательные методы
- `getRoot()`: Возвращает путь импорта модуля.
- `getNameFile()`: Стандартизирует имена файлов на основе компонента.
- `replaceOnce()`: Выполняет разовую замену для конкретной метки.
- `replacePropsValues()`, `replaceClassesValues()`, `replaceStylesValues()`: Преобразуют специфические типы свойств в оптимизированные списки или массивы.
- `isBoolean()`, `isNoMark()`: Вспомогательные методы для проверки типов и валидации меток.
- `getDefault()`: Извлекает значения по умолчанию для конкретных структурных элементов.
