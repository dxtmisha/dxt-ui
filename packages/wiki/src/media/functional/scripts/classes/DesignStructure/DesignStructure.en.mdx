# DesignStructure
The `DesignStructure` class is the primary orchestrator for processing structured components. It coordinates the reading of component data, the generation of CSS subclasses, and the transformation of tokens into styles, utilizing caching for performance optimization.

### Primary Methods
- `get()`: Retrieves the full structure data. If the data is not in the cache, it triggers the generation process via `DesignStructureRead`.
- `getClasses()`: Obtains a list of specific CSS subclasses defined within the component's structure.
- `getStyles()`: Returns an array of processed CSS styles derived from design tokens.
- `make()`: Triggers the actual generation of structure data and handles cache storage.

### Utility Methods
- `getDesign()`, `getDesignFirst()`: Return the design system name.
- `getComponentName()`, `getComponentNameFirst()`: Return the component name.
- `getFullComponentName()`: Combines design and component names.
- `getFileName()`: Returns the standardized file name for the component.
- `getPathName()`, `getPathClasses()`: Return cache file paths.
- `makeClasses()`: Generates data for component subclasses.
- `makeStyles()`: Performs the CSS style transformation.
