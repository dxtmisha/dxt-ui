# DesignStructure
Класс `DesignStructure` является основным оркестратором для обработки структурированных компонентов. Он координирует чтение данных компонента, генерацию CSS-подклассов и преобразование токенов в стили, используя кэширование для оптимизации производительности.

### Основные методы
- `get()`: Извлекает полные данные структуры. Если данных нет в кэше, запускает процесс генерации через `DesignStructureRead`.
- `getClasses()`: Получает список специфических CSS-подклассов, определенных в структуре компонента.
- `getStyles()`: Возвращает массив обработанных CSS-стилей, полученных из дизайн-токенов.
- `make()`: Запускает фактическую генерацию данных структуры и управляет сохранением в кэш.

### Вспомогательные методы
- `getDesign()`, `getDesignFirst()`: Возвращают название дизайн-системы.
- `getComponentName()`, `getComponentNameFirst()`: Возвращают название компонента.
- `getFullComponentName()`: Объединяет названия дизайна и компонента.
- `getFileName()`: Возвращает стандартизированное имя файла для компонента.
- `getPathName()`, `getPathClasses()`: Возвращают пути к файлам кэша.
- `makeClasses()`: Генерирует данные для подклассов компонента.
- `makeStyles()`: Выполняет преобразование CSS-стилей.
