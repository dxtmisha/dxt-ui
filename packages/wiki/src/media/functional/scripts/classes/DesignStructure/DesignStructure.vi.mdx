# DesignStructure
Lớp `DesignStructure` là bộ điều phối chính để xử lý các component có cấu trúc. Nó điều phối việc đọc dữ liệu component, tạo các lớp con CSS và chuyển đổi các token thành style, sử dụng cơ chế lưu trữ đệm (caching) để tối ưu hóa hiệu suất.

### Các phương thức chính
- `get()`: Lấy dữ liệu cấu trúc đầy đủ. Nếu dữ liệu không có trong cache, nó sẽ kích hoạt quy trình tạo thông qua `DesignStructureRead`.
- `getClasses()`: Lấy danh sách các lớp con CSS cụ thể được định nghĩa trong cấu trúc của component.
- `getStyles()`: Trả về một mảng các CSS style đã được xử lý từ các design token.
- `make()`: Kích hoạt việc tạo dữ liệu cấu trúc thực tế và xử lý việc lưu trữ vào cache.

### Các phương thức tiện ích
- `getDesign()`, `getDesignFirst()`: Trả về tên hệ thống thiết kế.
- `getComponentName()`, `getComponentNameFirst()`: Trả về tên component.
- `getFullComponentName()`: Kết hợp tên thiết kế và tên component.
- `getFileName()`: Trả về tên file tiêu chuẩn cho component.
- `getPathName()`, `getPathClasses()`: Trả về đường dẫn file cache.
- `makeClasses()`: Tạo dữ liệu cho các lớp con của component.
- `makeStyles()`: Thực hiện chuyển đổi CSS style.
