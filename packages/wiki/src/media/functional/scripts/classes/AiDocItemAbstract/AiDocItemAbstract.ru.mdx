# Класс AiDocItemAbstract

Абстрактный базовый класс, обеспечивающий логику автоматизированной генерации документации (файлов `.mdx`) и обновления исходного кода с помощью ИИ. Он координирует процесс сборки временных артефактов, подготовки контекста для ИИ и обработки полученных результатов.

## Ключевые особенности

- **Автоматизация Wiki** — создание и обновление файлов документации на основе анализа исходного кода.
- **Интеллектуальное обновление** — генерация описаний и модификация кода (например, добавление аннотаций) через ИИ.
- **Управление сборкой** — автоматический запуск `BuildItem` для анализа актуальных типов и структуры перед генерацией.
- **Синхронизация по дате** — генерация запускается только в случае, если исходный файл был изменен после последнего создания документации.

## Основной метод

### `make`

Запускает полный цикл работы с элементом документации. Метод проверяет актуальность файла, инициализирует сборку, готовит промпт для ИИ, отправляет запрос и распределяет полученные результаты по соответствующим файлам (код, MDX, описание).

**Основные этапы работы:**
1. Сравнение даты изменения исходного файла с датой существующего MDX.
2. Проверка доступности AI-инстанса.
3. Выполнение предварительной сборки (build) для извлечения актуальных интерфейсов.
4. Генерация контента через AI на основе подготовленного контекста.
5. Парсинг ответа (через разделитель `#########`) и запись данных в файлы.

**Возвращает:** `Promise<void>`

## Обработка данных

Методы для работы с содержимым и контекстом:

- `makeAi(): void` — Подготавливает подробный контекст для ИИ: передает актуальный код, демо-файлы и существующую документацию.
- `readPrompt(): string` — Читает и подготавливает шаблон промпта для выбранного языка документации.
- `initName(content: string): string` — Подставляет сгенерированный заголовок в текстовый шаблон.
- `readDemo(): string` — Загружает содержимое демо-файла для анализа примеров использования.

## Вспомогательные методы

- `isBuildDisabled(): boolean` — Проверяет наличие маркера `// wiki:build-none`, отключающего предварительную сборку.
- `getPathWiki(): string[]` — Генерирует массив путей для будущего файла MDX в структуре Wiki.
- `getItemDate(): Date` — Возвращает дату последней модификации исходного элемента.
- `getTitle(): string` — Вычисляет полный заголовок документации с учетом названия проекта и описания.

## Свойства

- `protected readonly build: BuildItem` — Менеджер сборки элемента.
- `protected readonly mdFile: ComponentWikiFile` — Файл документации MDX.
- `protected readonly code: ComponentWikiFile` — Исходный файл с кодом.
- `protected readonly ai?: AiAbstract` — Экземпляр ИИ для генерации.

## Точки расширения (Abstract)

Подклассы должны определить пути к специфичным для типа элемента ресурсам:
- `pathDemo: string[]` — Путь к файлу с демонстрацией использования.
- `pathPrompt: string[]` — Путь к файлу-шаблону промпта.

## Пример использования

```typescript
class MyCodeDoc extends AiDocItemAbstract {
  protected pathDemo = ['path', 'to', 'demo.ts'];
  protected pathPrompt = ['path', 'to', 'prompt.txt'];
}

const docGenerator = new MyCodeDoc(sourcePath, gitItem);
await docGenerator.make();
```
