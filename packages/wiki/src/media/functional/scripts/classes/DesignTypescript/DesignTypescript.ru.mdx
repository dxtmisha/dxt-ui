# DesignTypescript
Класс `DesignTypescript` предоставляет расширенные возможности для анализа исходного кода TypeScript с использованием официального TypeScript Compiler API. Он извлекает подробную информацию о типах, интерфейсах и классах, включая имена свойств, типы, значения по умолчанию и документацию JSDoc, что крайне важно для генерации точных спецификаций компонентов и поддержки IDE.

### Основные методы
- `get()`: Парсит целевой файл TypeScript и возвращает структурированный объект, содержащий всю извлеченную информацию о типах и свойствах.
- `make()`: Выполняет фактический обход абстрактного синтаксического дерева (AST) TypeScript для сбора данных.

### Вторичные методы
- `getDocumentation()`: Извлекает комментарии JSDoc, связанные с конкретным узлом кода.
- `getType()`: Разрешает и форматирует строку типа TypeScript для узла.
- `getDefault()`: Идентифицирует значения по умолчанию, назначенные свойствам или параметрам.
- `getPath()`: Возвращает абсолютный путь к анализируемому исходному файлу.
- `isType()`: Проверяет, представляет ли узел тип, который должен быть включен в анализ.
