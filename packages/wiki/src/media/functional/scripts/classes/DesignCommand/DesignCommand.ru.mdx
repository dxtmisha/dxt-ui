# DesignCommand
Класс `DesignCommand` — это абстрактный базовый класс, предназначенный для генерации и выполнения файлов сценариев, связанных с дизайн-системой. Он предоставляет стандартизированную среду для обработки операций с файлами, замены шаблонов и выполнения внешних команд во всех инструментах генерации, связанных с дизайном.

### Основные методы
- `make()`: Основной метод выполнения (абстрактный), который координирует процесс генерации для конкретных подклассов.
- `toReplace()`: Выполняет замену в шаблонах путем сопоставления значений свойств со структурными метками.
- `run()`: Выполняет внешнюю системную команду с помощью утилиты `execSync`.

### Вспомогательные методы
- `getPath()`: Возвращает абсолютный путь для генерации целевого файла.
- `getContent()`: Загружает содержимое файла шаблона.
- `writeFile()`: Безопасно записывает сгенерированный контент в целевой файл.
- `getReplace()`: Возвращает экземпляр `DesignReplace`, настроенный с текущими структурными данными.
- `toValue()`, `toValueState()`: Преобразуют структурные элементы в стандартизированные значения для шаблонов.
