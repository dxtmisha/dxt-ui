# DesignCommand
Lớp `DesignCommand` là một lớp cơ sở trừu tượng được thiết kế để tạo và thực thi các file script liên quan đến hệ thống thiết kế. Nó cung cấp một khung tiêu chuẩn để xử lý các thao tác file, thay thế template và thực thi các lệnh bên ngoài trong tất cả các công cụ tạo liên quan đến thiết kế.

### Các phương thức chính
- `make()`: Phương thức thực thi cốt lõi (trừu tượng) điều phối quy trình tạo cho các lớp con cụ thể.
- `toReplace()`: Thực hiện thay thế template bằng cách ánh xạ các giá trị thuộc tính với các marker cấu trúc.
- `run()`: Thực thi một lệnh hệ thống bên ngoài bằng tiện ích `execSync`.

### Các phương thức tiện ích
- `getPath()`: Trả về đường dẫn tuyệt đối cho việc tạo file đích.
- `getContent()`: Tải nội dung của file template.
- `writeFile()`: Ghi nội dung đã tạo vào file đích một cách an toàn.
- `getReplace()`: Trả về đối tượng `DesignReplace` được cấu hình với dữ liệu cấu trúc hiện tại.
- `toValue()`, `toValueState()`: Chuyển đổi các mục cấu trúc thành các giá trị tiêu chuẩn cho template.
