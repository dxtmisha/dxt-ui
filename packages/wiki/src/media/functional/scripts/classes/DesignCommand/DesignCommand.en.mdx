# DesignCommand
The `DesignCommand` class is an abstract base class designed for generating and executing script files related to the design system. It provides a standardized framework for handling file operations, template replacements, and external command execution across all design-related generation tools.

### Primary Methods
- `make()`: The core execution method (abstract) that orchestrates the generation process for specific subclasses.
- `toReplace()`: Performs template replacements by mapping property values to structural markers.
- `run()`: Executes an external system command using the `execSync` utility.

### Utility Methods
- `getPath()`: Returns the absolute path for the target file generation.
- `getContent()`: Loads the content of the template file.
- `writeFile()`: Safely writes the generated content to the target file.
- `getReplace()`: Returns the `DesignReplace` instance configured with the current structural data.
- `toValue()`, `toValueState()`: Transform structural items into standardized values for templates.
