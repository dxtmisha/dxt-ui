# LibraryExport Class

The `LibraryExport` class is responsible for aggregating and generating a unified export file for the library. It scans designated subfolders for TypeScript and Vue files, automatically compiling all exports into a single public interface to simplify library imports.

## Key Features

- **Unified Export Generation** ‚Äî Creates a comprehensive public entry point for all components, composables, functions, and classes.
- **Smart Module Detection** ‚Äî Automatically applies standard wildcard exports (`export *`) for TypeScript utilities and creates named exports for Vue SFC components.
- **Style Integration** ‚Äî Automatically checks for and integrates global CSS or SCSS stylesheets.
- **Intelligent Filtering** ‚Äî Excludes unit test files (`.test.`) and modules marked with the `// export:none` directive.

## Initialization

Initialize and execute the library export generator using the `make` method:

```typescript
import { LibraryExport } from '@dxtmisha/scripts'

const generator = new LibraryExport()
generator.make()
```

## Methods

### Core Execution

#### `make`
Triggers the main library compilation and entry point generation. It scans all configured subdirectories, collects the valid files, filters out test/none-export modules, and writes the unified import/export statements into the designated destination file.

**Parameters:**
- No parameters required.

**Returns:** `void` ‚Äî Modifies the target `library.ts` file on the filesystem and logs progress to the console.

```typescript
import { LibraryExport } from '@dxtmisha/scripts'

new LibraryExport().make()
```

### Helper Methods

- `getPath(directory: string): string[]` ‚Äî Returns the directory path components based on `UI_DIR_IN` and the specified directory name.
- `isExport(path: string | string[]): boolean` ‚Äî Evaluates if a file path is suitable for export by verifying it is not a test file and does not match `UI_FLAG_NOT_EXPORT`.
- `getDirectory(): LibraryFiles` ‚Äî Collects recursively all indexable files within the configured export folders and aggregates them.
- `getName(name: string): string` ‚Äî Converts the first character of the directory name to uppercase to standardize comments in the generated file.
- `getFile(path: string | string[]): string` ‚Äî Reads the file contents at the given path from the filesystem.
- `initFile(): string` ‚Äî Builds the raw content of the output library file, including imports, style setups, and wildcard/named exports.
- `initStyles(): string` ‚Äî Generates imports for global stylesheets (`style.scss` or `style.css`) if they are present in the root of the source directory.

## Dependencies and Configurations

The `LibraryExport` class interacts with several global configurations to perform structural generation:

- `UI_DIRS_LIST_EXPORT` ‚Äî Specifies the active folders that are scanned recursively for exports (`classes`, `components`, `composables`, `functions`, `global`, `types`).
- `UI_DIRS_FILE_EXPORT` ‚Äî Sets the destination output path of the compiled library file (`src/library.ts`).
- `UI_DIR_IN` ‚Äî Identifies the root source directory containing the modules (`src`).
- `UI_FLAG_NOT_EXPORT` ‚Äî Holds the regular expression matching the non-export directive (`/\/\/ *export:none/`).
