# Класс AiAbstract

Абстрактный базовый класс, обеспечивающий общую механику для интеграций с ИИ: накопление контекста (prompt), выбор модели и унифицированный цикл получения ответа. Он служит фундаментом для создания конкретных реализаций (например, для OpenAI, Google Gemini и др.).

## Ключевые особенности

- **Накопление контекста** — методы для постепенного сбора текста, изображений и других данных перед отправкой запроса.
- **Управление моделями** — возможность гибкого переключения между различными моделями в рамках одного инстанса.
- **Унифицированный цикл** — стандартизированный метод `generate`, скрывающий детали инициализации и вызова конкретных API.
- **Точки расширения** — четко определенные абстрактные методы для реализации специфики работы с конкретными провайдерами ИИ.

## Основные методы

### `generate`

Главный метод для получения ответа от ИИ. Он автоматически проверяет инициализацию клиента (вызывая `init()`, если это необходимо) и выполняет запрос к модели через защищенный метод `response`.

**Параметры:**
- `contents: string` — основной текст запроса или дополнительный контент, который будет добавлен к накопленному контексту.

**Возвращает:** `Promise<string>` — текстовый результат генерации.

## Методы накопления данных

Эти методы используются для формирования контекста перед вызовом генерации. Поддерживают чейнинг (за исключением `addPrompt`).

- `addImage(image: AiImageItem): this` — Добавляет изображение в список накопленных медиа-файлов.
- `addContent(content: string): this` — Добавляет текстовый блок в массив накопленного содержимого.
- `addPrompt(prompt: string): void` — Добавляет новую строку к базовому префиксу (системному промпту).

## Методы управления состоянием

Методы для очистки буферов и сброса накопленных данных:

- `resetImages(): this` — Полностью очищает список накопленных изображений.
- `resetContents(): this` — Очищает массив накопленных текстовых блоков.
- `resetPrompt(): void` — Обнуляет накопленный базовый промпт.

## Конфигурация

Методы для настройки рабочих параметров (поддерживают чейнинг):

- `setKey(key: string): this` — Устанавливает или изменяет API ключ. Если клиент уже был инициализирован, вызывает повторную инициализацию `init()`.
- `setModel(model: string): this` — Переключает используемую модель для генерации.

## Методы получения данных

- `getImages(): AiImageList` — Возвращает текущий список накопленных изображений.
- `getContents(): string[]` — Возвращает массив всех добавленных текстовых блоков.

## Точки расширения (Abstract)

Для создания своей интеграции необходимо наследоваться от `AiAbstract` и реализовать следующие методы:

- `init(): void` — Логика инициализации клиента, аутентификации и настройки транспорта.
- `response(model: string, contents: string): Promise<string>` — Непосредственный вызов API провайдера для получения текста.
- `toImages(): void` — Преобразование накопленных изображений во внутренний формат провайдера.
- `toContents(): void` — Преобразование накопленного контента во внутренний формат провайдера.

## Пример использования

```typescript
import { AiAbstract } from '@dxtmisha/scripts'

class MyAiProvider extends AiAbstract<SomeApiClient> {
  protected init() {
    this.ai = new SomeApiClient({ apiKey: this.key });
  }

  protected async response(model: string, contents: string) {
    const result = await this.ai.chat.completions.create({ model, prompt: contents });
    return result.choices[0].text;
  }
  
  protected toImages() { /* ... */ }
  protected toContents() { /* ... */ }
}

const ai = new MyAiProvider('api-key', 'gpt-4');

// Постепенное формирование запроса
ai.addPrompt('Ты — профессиональный переводчик.')
  .addContent('Переведи следующий текст на английский:')
  .addImage({ url: 'image.png', type: 'image/png' });

const translation = await ai.generate('Привет, как дела?');
```
