# Класс ComponentWiki

Сложный класс-оркестратор, отвечающий за автоматическое создание и обновление Wiki-артефактов компонента (MDX-документации, Storybook-историй и типизации). Он объединяет процессы временной сборки проекта и анализа кода через ИИ.

## Назначение

Класс автоматизирует написание документации для Vue-компонентов. Он анализирует реальный код компонента, "показывает" его искусственному интеллекту и на основе полученных данных генерирует структурированную документацию, примеры использования (stories) и файлы типов.

## Основной метод

### `make`

Запускает комплексный процесс подготовки и генерации документации.

**Алгоритм работы:**
1. Инициализация временного билда компонента (`BuildItem`).
2. Запуск сборки для извлечения исполняемого кода.
3. Подготовка контекста для ИИ через `makeAi` (сбор исходного кода, типов и существующих MDX).
4. Вызов ИИ-генератора через `aiGenerate`.
5. Разбор ответа ИИ и обновление соответствующих файлов (`codeFile`, `typesFile`, `storiesFile`, `mdFile`).
6. Очистка временных ресурсов сборки.

**Параметры:**
- Метод не принимает аргументов.

**Возвращает:** `void`

## Методы подготовки и генерации

- `makeAi(): void` — Собирает все необходимые данные (код, типы, демо-MDX) и добавляет их в промпт ИИ.
- `aiGenerate(): Promise<void>` — Выполняет запрос к ИИ, получает результат и распределяет его по файлам проекта, используя разделители и метки (`MARK_FULL`).
- `readPrompt(): string` — Читает системный шаблон инструкций для ИИ и подставляет текущий язык Wiki.

## Навигация и пути

- `getRootComponent(): string[]` — Определяет корневую директорию компонента.
- `getPathWiki(): string[]` — Определяет путь к папке `_wiki` внутри компонента.
- `getName(): string` — Получает чистое имя компонента.

## Пример использования

```typescript
import { ComponentWiki } from '@dxtmisha/scripts'

// Путь к компоненту относительно папки компонентов
const wiki = new ComponentWiki('Basic/Button');

// Запуск процесса автоматической документации
wiki.make();
```
