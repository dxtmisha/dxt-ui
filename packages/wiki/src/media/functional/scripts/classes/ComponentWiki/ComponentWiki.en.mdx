# ComponentWiki Class

A complex orchestrator class responsible for the automated creation and updating of component Wiki artifacts (MDX documentation, Storybook stories, and typing). It combines temporary project builds with AI-driven code analysis.

## Purpose

The class automates the writing of documentation for Vue components. It analyzes the actual component code, "presents" it to the AI, and based on the retrieved data, generates structured documentation, usage examples (stories), and type files.

## Primary Method

### `make`

Triggers the comprehensive process of documentation preparation and generation.

**Execution Algorithm:**
1. Initializing a temporary build of the component (`BuildItem`).
2. Running the build to extract executable code.
3. Preparing the AI context via `makeAi` (collecting source code, types, and existing MDX).
4. Invoking the AI generator via `aiGenerate`.
5. Parsing the AI response and updating the corresponding files (`codeFile`, `typesFile`, `storiesFile`, `mdFile`).
6. Cleaning up temporary build resources.

**Parameters:**
- This method takes no arguments.

**Returns:** `void`

## Preparation and Generation Methods

- `makeAi(): void` — Aggregates all required data (code, types, demo MDX) and appends it to the AI prompt.
- `aiGenerate(): Promise<void>` — Performs the AI request, receives the result, and distributes it across project files using delimiters and markers (`MARK_FULL`).
- `readPrompt(): string` — Reads the system instruction template for the AI and injects the current Wiki language.

## Navigation and Paths

- `getRootComponent(): string[]` — Determines the component's root directory.
- `getPathWiki(): string[]` — Determines the path to the `_wiki` folder within the component.
- `getName(): string` — Retrieves the clean component name.

## Usage Example

```typescript
import { ComponentWiki } from '@dxtmisha/scripts'

// Path to the component relative to the components folder
const wiki = new ComponentWiki('Basic/Button');

// Starting the automated documentation process
wiki.make();
```
