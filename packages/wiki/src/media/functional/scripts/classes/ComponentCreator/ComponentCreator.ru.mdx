# Класс ComponentCreator

Класс, предназначенный для автоматизации процесса создания новых компонентов. Он сканирует дерево директорий компонентов и автоматически создает необходимые файлы-заготовки в пустых папках.

## Назначение

Класс упрощает инициализацию новых компонентов в проекте. Вместо ручного копирования файлов, разработчик может просто создать папку с именем компонента, и `ComponentCreator` наполнит её базовой структурой (кодом, типами, документацией) на основе эталонных шаблонов.

## Основной метод

### `make`

Основной метод, запускающий процесс сканирования и создания. Он находит все пустые директории в целевом пути компонентов и для каждой из них запускает механизм создания файлов.

**Алгоритм работы:**
1. Получение списка всех директорий внутри `UI_DIRS_COMPONENTS` рекурсивно.
2. Фильтрация списка: оставляются только те папки, в которых полностью отсутствуют файлы компонентов (проверяется через `getComponentPaths`).
3. Итерация по отфильтрованным путям.
4. Создание экземпляра `ComponentItem` для каждого пути и вызов его метода `make()`.

**Параметры:**
- Метод не принимает аргументов.

**Возвращает:** `void`

## Вспомогательные методы

- `getDirs(): string[]` — Возвращает массив путей к пустым директориям компонентов, требующим инициализации.

## Пример использования

```typescript
import { ComponentCreator } from '@dxtmisha/scripts'

const creator = new ComponentCreator();

// Сканирование и создание заготовок в пустых папках
creator.make();
```
