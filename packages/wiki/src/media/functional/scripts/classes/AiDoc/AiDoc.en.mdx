# AiDoc Class

The orchestrator class for generating AI-driven documentation. It reads configuration settings, scans target directories, and coordinates the processing of files through the AI documentation pipeline.

## Key Features

- **Multi-Directory Processing** — Batch processes documentation for multiple configured source directories.
- **Config-Driven Automation** — Automatically retrieves target paths and directories from `PropertiesConfig`.
- **Error Resilience** — Handles processing errors per file, ensuring a single failure doesn't halt the entire pipeline (unless critical).
- **Git Awareness** — Utilizes `GitRead` to filter and identify files within target directories.

## Primary Method

### `make`

The entry point for the documentation generation process. It retrieves the list of target directories and the base Storybook path from the configuration, then iterates through each directory to process its files.

**Main workflow steps:**
1. Fetches the list of directories to be documented via `PropertiesConfig`.
2. Logs the target Storybook output path.
3. Iteratively calls `makeDirectory` for each configured path.
4. Provides high-level logging for the overall process.

**Returns:** `Promise<void>`

## Data Processing Methods

- `makeDirectory(dir: string): Promise<void>` — Processes all valid files within a specific directory. Iterates through the file list and triggers the AI generation logic for each item via `AiDocType.make`.
- `protected getListByDirectory(dir: string): string[]` — Retrieves and filters the list of files in a directory using `GitRead`, ensuring only relevant files are processed.

## Usage Example

```typescript
import { AiDoc } from '@dxtmisha/scripts'

const orchestrator = new AiDoc();

// Start the documentation generation process
await orchestrator.make();
```
