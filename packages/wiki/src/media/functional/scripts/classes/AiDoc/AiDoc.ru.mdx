# Класс AiDoc

Центральный движок для координации процесса генерации AI-документации во всем проекте. Он управляет чтением конфигурации, сканированием директорий и делегированием задач по документированию специализированным обработчикам ИИ.

## Ключевые особенности

- **Пакетная обработка** — автоматически перебирает настроенные директории для генерации документации сразу для множества файлов.
- **Интеграция на уровне проекта** — считывает настройки проекта через `PropertiesConfig` для определения целевых путей и областей охвата.
- **Интеграция с Git** — использует `GitRead` для идентификации релевантных файлов и их фильтрации на основе статуса репозитория.
- **Устойчивость к ошибкам** — надежная обработка исключений в процессе документирования для обеспечения стабильной работы или четкого сообщения о проблемах.

## Инициализация

Класс `AiDoc` обычно инициализируется как основной координатор для запуска процесса сборки документации.

```typescript
import { AiDoc } from '@dxtmisha/scripts'

const engine = new AiDoc()

// Запуск обработки всех настроенных директорий
await engine.make()
```

## Методы

### Управление

- `make(): Promise<void>` — основная точка входа. Считывает конфигурацию и запускает обработку всех определенных директорий документации.
- `makeDirectory(dir: string): Promise<void>` — обрабатывает все соответствующие файлы внутри конкретной директории, делегируя задачи специализированным обработчикам ИИ.


Высокоуровневый координатор, который не предоставляет открытых свойств конфигурации или сложного внутреннего состояния.
