# Класс AiDoc

Основной управляющий класс для процесса автоматизированной генерации документации с использованием ИИ. Он считывает настройки конфигурации, сканирует целевые директории и координирует обработку файлов через конвейер AI-генерации.

## Ключевые особенности

- **Мульти-директориальная обработка** — пакетная генерация документации для нескольких настроенных исходных папок.
- **Автоматизация на основе конфига** — автоматическое получение целевых путей и директорий из `PropertiesConfig`.
- **Устойчивость к ошибкам** — индивидуальная обработка ошибок для каждого файла, предотвращающая остановку всего процесса при локальных сбоях.
- **Интеграция с Git** — использование `GitRead` для эффективной фильтрации и поиска файлов в целевых директориях.

## Основной метод

### `make`

Точка входа в процесс генерации документации. Метод извлекает список целевых директорий и базовый путь Storybook из конфигурации, после чего запускает цикл обработки для каждой директории.

**Основные этапы работы:**
1. Получение списка директорий для документирования через `PropertiesConfig`.
2. Логирование целевого пути вывода в Storybook.
3. Последовательный вызов `makeDirectory` для каждого настроенного пути.
4. Высокоуровневое логирование общего прогресса.

**Возвращает:** `Promise<void>`

## Методы обработки данных

- `makeDirectory(dir: string): Promise<void>` — Обрабатывает все валидные файлы в конкретной папке. Проходит по списку файлов и запускает логику генерации для каждого элемента через `AiDocType.make`.
- `protected getListByDirectory(dir: string): string[]` — Получает и фильтрует список файлов в директории с помощью `GitRead`, гарантируя обработку только актуальных файлов.

## Пример использования

```typescript
import { AiDoc } from '@dxtmisha/scripts'

const orchestrator = new AiDoc();

// Запуск процесса генерации документации
await orchestrator.make();
```
