# DesignStructureStyles Class

The `DesignStructureStyles` class is responsible for converting a component's structural design tokens into a standardized array of CSS property strings. By leveraging the `StylesProperties` engine, it ensures that every component inherits its base visual identity directly from the design system's central theme configuration.

## Key Features

- **Token-to-CSS Transformation** — Automatically maps abstract design tokens to concrete CSS properties.
- **BEM Selector Scoping** — Wraps generated styles in a component-level selector (`& { ... }`) to ensure style isolation.
- **Hierarchical Inheritance** — Utilizes parent structural metadata to resolve nested property variables accurately.
- **Standardized Formatting** — Applies consistent indentation and line breaks, ensuring generated style files remain readable.

## Initialization

To initialize the style compiler, provide the identifier of the component whose styles needs to be generated. The class will automatically load the relevant structural metadata.

**Parameters:**
- `component: string` — The identifier of the component (e.g., `'button'`).

```typescript
import { DesignStructureStyles } from '@dxtmisha/scripts'

const compiler = new DesignStructureStyles('button')
const stylesArray = compiler.get()
```

## Methods

### Control

- `make(): string[]` — Triggers the transformation process and returns the array of CSS property strings.

## Operations

### The Style Compilation Flow

When the compiler is initialized, it follows a structured pipeline:
1. **Metadata Loading**: It uses the parent `DesignStructureItemAbstract` logic to fetch the specific `PropertyItemsItem` associated with the component.
2. **Contextual Selection**: The engine wraps all output properties within a self-referencing CSS selector (`& { ... }`) to maintain compatibility with CSS-in-JS and preprocessors.
3. **Property Resolution**: It instantiates `StylesProperties`, passing the collected metadata. This sub-engine then traverses every token, resolving theme variables, units, and fallbacks.
4. **Output Buffering**: The final list of properties is joined into a flat array of strings, ready for injection into a CSS file or a style tag.

### Integration with Theme Engine

This class acts as the bridge between raw design data and the final visual representation. By centralizing the style compilation logic, the system ensures that changes to the theme's core architecture (such as variable naming conventions or unit logic) are automatically propagated to all components during the next build cycle.
