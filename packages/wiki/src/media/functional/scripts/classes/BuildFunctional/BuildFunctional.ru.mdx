# Класс BuildFunctional

Класс, отвечающий за автоматическую генерацию файла входной точки (entry point) для функциональных утилит библиотеки. Он обеспечивает корректный ре-экспорт всех доступных функций из пакета `@dxtmisha/functional`.

## Назначение

Класс используется в процессе сборки библиотеки для синхронизации состава доступных утилит. Он динамически считывает все экспорты из основного пакета функционала и формирует типизированный файл `functional.ts`, который служит "фасадом" для доступа к утилитам.

## Основной метод

### `make`

Основной метод, выполняющий генерацию файла. Он собирает список всех доступных функций, формирует код ре-экспорта и записывает результат в файловую систему.

**Алгоритм работы:**
1. Получение полного списка экспортируемых имен из модуля `@dxtmisha/functional`.
2. Подготовка заголовка файла с экспортом типов (`export type *`).
3. Сортировка списка имен по алфавиту для стабильности сборки.
4. Итерация по списку и создание строк индивидуального ре-экспорта для каждой функции.
5. Запись итогового кода в файл `functional.ts` по пути, указанному в `UI_DIRS_LIBRARY`.

**Параметры:**
- Метод не принимает аргументов.

**Возвращает:** `void`

## Вспомогательные методы

- `getList(): string[]` — Возвращает массив имен (ключей) всех экспортируемых объектов из пакета `@dxtmisha/functional`.

## Пример использования

```typescript
import { BuildFunctional } from '@dxtmisha/scripts'

const builder = new BuildFunctional();

// Генерация файла функционального входа
builder.make();
```
