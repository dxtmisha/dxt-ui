# BuildFunctional Class

A class responsible for automatically generating the entry point file for the library's functional utilities. It ensures proper re-exporting of all available functions from the `@dxtmisha/functional` package.

## Purpose

The class is used during the library's build process to synchronize the set of available utilities. It dynamically reads all exports from the core functional package and generates a typed `functional.ts` file, which serves as a "facade" for accessing the utilities.

## Primary Method

### `make`

The main method that performs the file generation. It compiles a list of all available functions, formats the re-export code, and writes the result to the file system.

**Execution Algorithm:**
1. Retrieving the full list of exported names from the `@dxtmisha/functional` module.
2. Preparing the file header with type exports (`export type *`).
3. Sorting the list of names alphabetically for build stability.
4. Iterating through the list and creating individual re-export lines for each function.
5. Writing the resulting code to the `functional.ts` file at the path specified in `UI_DIRS_LIBRARY`.

**Parameters:**
- This method takes no arguments.

**Returns:** `void`

## Helper Methods

- `getList(): string[]` — Returns an array of names (keys) for all exported objects from the `@dxtmisha/functional` package.

## Usage Example

```typescript
import { BuildFunctional } from '@dxtmisha/scripts'

const builder = new BuildFunctional();

// Generating the functional entry file
builder.make();
```
