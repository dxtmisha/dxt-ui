# Класс AiDocType

Статический класс-менеджер, отвечающий за определение типа объекта и выбор соответствующего генератора документации ИИ. Он служит центральным узлом для распределения задач между различными реализациями `AiDocItemAbstract`.

## Ключевые особенности

- **Автоматическое определение типа** — класс анализирует метаданные файла (`GitFileItem`) и подбирает нужный тип документации.
- **Масштабируемость** — поддержка добавления новых типов документируемых элементов через метод `add`.
- **Централизованное управление** — инкапсуляция логики выбора между генераторами для классов, функций, композитных функций и других элементов.

## Основной метод

### `make`

Статический метод, который является связующим звеном между списком файлов и конкретными генераторами. Он определяет, к какому типу относится переданный элемент, создает экземпляр соответствующего класса (например, `AiDocItemClasses` или `AiDocItem`) и запускает процесс генерации.

**Алгоритм работы:**
1. Получение целевого пути Storybook из конфигурации.
2. Итерация по списку зарегистрированных типов (`aiDocTypes`).
3. Вызов функции `check` у каждого типа для проверки соответствия элемента.
4. При нахождении соответствия: создание экземпляра генератора и вызов его метода `make()`.
5. Прекращение поиска после первого успешного совпадения.

**Параметры:**
- `item: GitFileItem` — объект с данными о файле из Git.

**Возвращает:** `Promise<void>`

## Обработка типов

- `static add(type: AiDocTypeItem): void` — Позволяет динамически регистрировать новые типы документации, добавляя их в начало списка приоритетов.

## Свойства

- `protected static types: AiDocTypeItem[]` — Список доступных типов документации, инициализируемый стандартным набором `aiDocTypes`.

## Пример использования

```typescript
import { AiDocType } from '@dxtmisha/scripts'

// Генерация документации для конкретного файла
await AiDocType.make(gitFileItem);
```
