# AiDocType Class

A static manager class responsible for determining the object type and selecting the appropriate AI documentation generator. It serves as a central hub for distributing tasks among different implementations of `AiDocItemAbstract`.

## Key Features

- **Automatic Type Detection** — the class analyzes file metadata (`GitFileItem`) and selects the appropriate documentation type.
- **Scalability** — supports adding new types of documented elements via the `add` method.
- **Centralized Management** — encapsulates the logic for choosing between generators for classes, functions, composables, and other elements.

## Primary Method

### `make`

A static method that serves as a bridge between the file list and specific generators. It determines the type of the passed element, creates an instance of the corresponding subclass (e.g., `AiDocItemClasses` or `AiDocItem`), and starts the generation process.

**Execution Algorithm:**
1. Retrieves the target Storybook path from the configuration.
2. Iterates through the list of registered types (`aiDocTypes`).
3. Calls the `check` function for each type to verify the element's compliance.
4. Upon finding a match: creates a generator instance and calls its `make()` method.
5. Terminates the search after the first successful match.

**Parameters:**
- `item: GitFileItem` — the file data object from Git.

**Returns:** `Promise<void>`

## Type Handling

- `static add(type: AiDocTypeItem): void` — Allows dynamic registration of new documentation types, adding them to the beginning of the priority list.

## Properties

- `protected static types: AiDocTypeItem[]` — The list of available documentation types, initialized with the standard `aiDocTypes` set.

## Usage Example

```typescript
import { AiDocType } from '@dxtmisha/scripts'

// Generating documentation for a specific file
await AiDocType.make(gitFileItem);
```
