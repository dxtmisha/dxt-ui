# DesignUi
Класс `DesignUi` является оркестратором верхнего уровня для всего процесса генерации и управления компонентами пользовательского интерфейса. Он координирует создание отдельных компонентов через `DesignComponent`, управляет генерацией на основе конструкторов через `DesignConstructors` и гарантирует, что конфигурации на уровне проекта, такие как `package.json`, правильно обновлены для отражения доступных компонентов и их экспорта.

### Основные методы
- `make()`: Основная точка входа, которая запускает последовательность задач генерации: обработку конструкторов, создание отдельных компонентов и обновление экспортов пакета.
- `makeForce()`: Запускает полную принудительную регенерацию всех компонентов и их конфигураций.

### Вспомогательные методы
- `makeComponents()`: Выполняет обход библиотеки компонентов и запускает генерацию каждого компонента.
- `makePackage()`: Обновляет файл `package.json` актуальной информацией о компонентах, такой как версия и экспортируемые модули.
- `getList()`: Сканирует директорию дизайн-системы для поиска валидных папок компонентов.
- `getPath()`: Возвращает абсолютный путь к корневой директории дизайн-системы.
- `getPackagePath()`: Возвращает абсолютный путь к основному файлу `package.json`.
