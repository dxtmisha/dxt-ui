# DesignComponent
Класс `DesignComponent` — это комплексный оркестратор для генерации отдельных компонентов. Он управляет всем процессом создания и обновления файлов, специфичных для компонента, включая основную логику компонента, внутренние свойства, общедоступные интерфейсы свойств, стили и начальную документацию, гарантируя, что все части соответствуют стандартам проекта.

### Основные методы
- `make()`: Метод верхнего уровня, запускающий полный цикл генерации/обновления компонента.
- `makeForce()`: Принудительно запускает полную регенерацию, при необходимости перезаписывая существующие файлы.
- `makeConfig()`: Генерирует основной файл конфигурации и логики для компонента.
- `makeProperties()`: Генерирует файл внутреннего управления свойствами.
- `makeProps()`: Генерирует публичный интерфейс свойств.
- `makeStyles()`: Генерирует определения стилей компонента.
- `makeWiki()`: Инициализирует или обновляет страницы вики-документации компонента.

### Вторичные методы
- `getDesign()`, `getComponentName()`, `getFileName()`: Извлекают идентифицирующие имена компонента.
- `getPath()`, `getPathFile()`: Разрешают целевые пути файлов для генерации.
- `getContent()`: Загружают содержимое шаблона для конкретных типов файлов.
- `toReplace()`: Настраивают заполнители шаблона структурными данными компонента.
- `toWrite()`: Сохраняют сгенерированный контент в файловую систему.
