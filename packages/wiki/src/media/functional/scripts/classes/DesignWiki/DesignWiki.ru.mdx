# DesignWiki
Класс `DesignWiki` отвечает за управление генерацией и инициализацией файлов описания вики для компонентов. Он гарантирует, что каждый компонент имеет необходимые файлы `.mdx` для документации на всех поддерживаемых языках, предотвращая дублирование и поддерживая согласованную структуру.

### Основные методы
- `make()`: Основная точка входа, которая проверяет наличие файлов вики и запускает создание недостающих с помощью сервиса `ComponentWiki`.
- `makeForce()`: Принудительно запускает регенерацию или перезапись существующих файлов вики.

### Методы конфигурации
- `is()`: Проверяет, существует ли директория компонента и содержит ли она файл описания.
- `getDescriptionPath()`: Возвращает абсолютный путь к основному файлу описания компонента.
- `getPath()`: Возвращает базовый путь, где расположена документация компонентов.
