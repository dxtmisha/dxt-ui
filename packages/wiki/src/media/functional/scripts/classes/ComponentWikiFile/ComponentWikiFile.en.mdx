# ComponentWikiFile Class

A utility class for atomic reading and writing of individual documentation and code files within the Wiki system. It ensures the safety of write operations and provides metadata support (e.g., update dates).

## Key Features

- **Safe Writing** — content is written only if it is non-empty and differs from the current version.
- **Version History** — supports saving the previous version of the file with the `__old.txt` suffix.
- **Automated Dating** — capability to automatically add or update the `**Date: ...**` marker in the file body.
- **Smart Formatting** — automatic cleanup of Markdown code blocks (\`\`\`ts) when writing to code files.

## Primary Method

### `write`

Writes the passed content to the file, applying formatting and date update rules.

**Execution Algorithm:**
1. Verifying that the content contains data.
2. Cleaning up Markdown wrappers if present.
3. Comparing with the current file content.
4. If data is changed: creating a backup (`isOld`) and writing the new content with an updated date (`addDate`).

**Parameters:**
- `content: string` — the text to write.

**Returns:** `void`

## Information Retrieval Methods

- `getPath(): string` — Returns the full absolute path to the file.
- `read(): string` — Reads and returns the file content (or an empty string if not found).
- `getDate(): Datetime` — Extracts the update date from the file content.

## Processing Methods

- `addDate(content: string): string` — Inserts or replaces the line with the last update date at the end of the file.
- `extractDateFromContent(content: string): Datetime` — Searches for the date string using a regular expression in the text.

## Usage Example

```typescript
import { ComponentWikiFile } from '@dxtmisha/scripts'

const file = new ComponentWikiFile(['path', 'to', 'file.mdx'], true);

// Reading and automatically updating content with a date
const content = file.read();
file.write(content + '\nNew section');
```
