# Класс ComponentWikiFile

Служебный класс для атомарного чтения и записи отдельных файлов документации и кода в рамках Wiki-системы. Он обеспечивает безопасность операций записи и поддержку метаданных (например, дат обновления).

## Ключевые особенности

- **Безопасная запись** — контент записывается только если он не пустой и отличается от текущего.
- **История изменений** — поддержка сохранения предыдущей версии файла с суффиксом `__old.txt`.
- **Автоматическое датирование** — возможность автоматического добавления или обновления метки `**Date: ...**` в теле файла.
- **Умное форматирование** — автоматическая очистка Markdown-блоков кода (\`\`\`ts) при записи в файлы кода.

## Основной метод

### `write`

Записывает переданное содержимое в файл, применяя правила форматирования и обновления даты.

**Алгоритм работы:**
1. Проверка контента на наличие данных.
2. Очистка Markdown-оберток, если они присутствуют.
3. Сравнение с текущим содержимым файла.
4. Если данные изменились: создание резервной копии (`isOld`) и запись нового контента с обновленной датой (`addDate`).

**Параметры:**
- `content: string` — текст для записи.

**Возвращает:** `void`

## Методы получения информации

- `getPath(): string` — Возвращает полный абсолютный путь к файлу.
- `read(): string` — Читает и возвращает содержимое файла (или пустую строку, если файл не найден).
- `getDate(): Datetime` — Извлекает дату обновления из содержимого файла.

## Терминальные методы

- `addDate(content: string): string` — Вставляет или заменяет строку с датой последнего обновления в конце файла.
- `extractDateFromContent(content: string): Datetime` — Ищет строку даты по регулярному выражению в тексте.

## Пример использования

```typescript
import { ComponentWikiFile } from '@dxtmisha/scripts'

const file = new ComponentWikiFile(['path', 'to', 'file.mdx'], true);

// Чтение и автоматическое обновление контента с датой
const content = file.read();
file.write(content + '\nНовый раздел');
```
