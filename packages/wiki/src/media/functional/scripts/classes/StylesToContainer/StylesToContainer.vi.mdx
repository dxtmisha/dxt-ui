# Lớp StylesToContainer

Lớp `StylesToContainer` chịu trách nhiệm tạo các truy vấn CSS `@container` từ các thuộc tính thiết kế. Nó triển khai logic để bao bọc các quy tắc style lồng nhau bên trong một khối container query.

## Các tính năng chính

- **Bao bọc Container Query** — Tự động bao bọc nội dung style được tạo bên trong một quy tắc `@container`.
- **Phân giải truy vấn động** — Sử dụng giá trị thuộc tính thiết kế để định nghĩa điều kiện của container query.
- **Quản lý thụt lề** — Xử lý chính xác thụt lề khối và các dòng trống dẫn đầu tùy chọn.

## Khởi tạo

Khởi tạo trình chuyển đổi `StylesToContainer` với thuộc tính bắt buộc và một hàm gọi nội dung cho các quy tắc lồng nhau.

**Tham số:**
- `property: PropertyItemsItem` — nhánh hiện tại của cây thuộc tính đang được xử lý.
- `space: string` — chuỗi thụt lề (ví dụ: `"  "` hoặc `"\t"`).
- `content?: () => string[]` — một hàm có thể gọi trả về một mảng các chuỗi đại diện cho các quy tắc style nội bộ.
- `first?: boolean` — cho biết đây có phải là phần tử đầu tiên trong cấp độ hiện tại hay không (tùy chọn, mặc định: `true`).

```typescript
import { StylesToContainer } from '@dxtmisha/scripts'

// Sử dụng trong logic chuyển đổi
const converter = new StylesToContainer(property, '  ', contentCallable)
const result = converter.make()
```

## Các phương thức

### Điều khiển

- `make(): string[]` — Điểm bắt đầu chính. Điều phối việc chuyển đổi và thêm thụt lề cần thiết vào mỗi dòng.
