# DesignStructureRead
Класс `DesignStructureRead` отвечает за глубокий анализ и парсинг свойств зависимостей компонента. Он обрабатывает необработанные данные токенов для идентификации состояний компонента, доступных значений и свойств, подходящих для публичных props, гарантируя готовность структурированных данных для генерации кода.

### Основные методы
- `get()`: Основная точка входа, координирующая весь процесс чтения и трансформации.
- `makeMain()`: Сканирует свойства для определения тех, которые должны быть включены в публичный интерфейс свойств компонента.
- `makeState()`: Анализирует значения свойств для определения состояний компонента (например, hover, active, варианты размеров).
- `toValue()`: Нормализует необработанные значения свойств в стандартизированный формат для компонентов.

### Вторичные методы
- `getStates()`: Возвращает список идентифицированных состояний компонента.
- `isProps()`: Проверяет, должно ли конкретное свойство быть доступно как публичный prop.
- `isLinkClass()`: Проверяет, представляет ли значение свойства ссылку на CSS-класс.
- `getClass()`, `getClassState()`: Управляют разрешением имен классов из ссылок.
- `makeValue()`, `makeValueAll()`, `makeValueUnique()`, `makeValueState()`: Внутренние утилиты для обработки и оптимизации значений свойств в различных состояниях.
- `toName()`: Форматирует идентификаторы свойств в человекочитаемые или удобные для кода имена.
