# DesignStructureRead
The `DesignStructureRead` class is responsible for the deep analysis and parsing of a component's dependency properties. it processes raw token data to identify component states, available values, and properties suitable for public props, ensuring that the structured data is ready for code generation.

### Primary Methods
- `get()`: Main entry point that orchestrates the entire reading and transformation process.
- `makeMain()`: Scans properties to identify which should be included in the component’s public property interface.
- `makeState()`: Analyzes property values to determine component states (e.g., hover, active, size variants).
- `toValue()`: Normalizes raw property values into a standardized format for components.

### Secondary Methods
- `getStates()`: Returns the list of identified component states.
- `isProps()`: Validates if a specific property should be exposed as a public prop.
- `isLinkClass()`: Checks if a property value represents a reference to a CSS class.
- `getClass()`, `getClassState()`: Handle class name resolution from references.
- `makeValue()`, `makeValueAll()`, `makeValueUnique()`, `makeValueState()`: Internal utilities for processing and optimizing property values across states.
- `toName()`: Formats property identifiers into human-readable or code-friendly names.
