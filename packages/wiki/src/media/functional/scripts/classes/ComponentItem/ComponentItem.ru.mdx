# Класс ComponentItem

Класс, отвечающий за физическое создание файлов конкретного компонента на основе шаблонов. Он выполняет чтение эталонных файлов и замену плейсхолдеров на реальные данные компонента.

## Назначение

`ComponentItem` инкапсулирует логику трансформации шаблонов в готовый код. Он обрабатывает пути, имена и метаданные проекта, чтобы сгенерированные файлы (Vue, TS, MDX) сразу соответствовали контексту создаваемого компонента.

## Основной метод

### `make`

Запускает процесс генерации файлов для компонента. Метод последовательно обрабатывает каждый файл из директории шаблонов.

**Алгоритм работы:**
1. Получение списка всех файлов из директории шаблонов `media/templates/componentDoc`.
2. Итерация по списку файлов.
3. Чтение содержимого каждого шаблона.
4. Применение трансформаций (замена плейсхолдеров через `replacement`).
5. Запись итогового контента в целевую директорию компонента с сохранением структуры.

**Параметры:**
- Метод не принимает аргументов.

**Возвращает:** `void`

## Методы трансформации и работы с путями

- `getName(): string` — Извлекает имя компонента из пути его директории.
- `replacement(contentOrPath: string): string` — Заменяет плейсхолдеры (например, `ComponentDoc`, `[project]`, `[path]`) на реальные значения.
- `writeFile(path: string, content: string): void` — Записывает файл в целевую папку и устанавливает необходимые права доступа.

## Вспомогательные методы доставки данных

- `getFilesSample(): string[]` — Возвращает список всех файлов в директории шаблонов.
- `getProjectName(): string` — Получает имя проекта из `package.json`.
- `readPackage(): Record<string, any>` — Загружает данные из файла конфигурации пакета.

## Пример использования

```typescript
import { ComponentItem } from '@dxtmisha/scripts'

// Путь к новой директории компонента
const item = new ComponentItem('path/to/MyComponent');

// Генерация файлов из шаблонов
item.make();
```
