# ComponentItem Class

A class responsible for the physical creation of files for a specific component based on templates. It performs the reading of reference files and replaces placeholders with actual component data.

## Purpose

`ComponentItem` encapsulates the logic for transforming templates into ready-to-use code. It handles paths, names, and project metadata so that the generated files (Vue, TS, MDX) immediately align with the context of the component being created.

## Primary Method

### `make`

Triggers the file generation process for the component. The method sequentially processes each file from the template directory.

**Execution Algorithm:**
1. Retrieving a list of all files from the template directory `media/templates/componentDoc`.
2. Iterating through the file list.
3. Reading the content of each template.
4. Applying transformations (replacing placeholders via `replacement`).
5. Writing the resulting content to the target component directory while maintaining the structure.

**Parameters:**
- This method takes no arguments.

**Returns:** `void`

## Transformation and Path Methods

- `getName(): string` — Extracts the component name from its directory path.
- `replacement(contentOrPath: string): string` — Replaces placeholders (e.g., `ComponentDoc`, `[project]`, `[path]`) with real values.
- `writeFile(path: string, content: string): void` — Writes the file to the target folder and sets the required access permissions.

## Data Delivery Helper Methods

- `getFilesSample(): string[]` — Returns a list of all files in the template directory.
- `getProjectName(): string` — Retrieves the project name from `package.json`.
- `readPackage(): Record<string, any>` — Loads data from the package configuration file.

## Usage Example

```typescript
import { ComponentItem } from '@dxtmisha/scripts'

// Path to the new component directory
const item = new ComponentItem('path/to/MyComponent');

// Generating files from templates
item.make();
```
