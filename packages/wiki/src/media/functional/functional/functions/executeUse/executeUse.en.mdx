# `executeUse`

`executeUse` is a factory that wraps any initializer function and turns its result into a **managed singleton**. The object is created exactly once, frozen (`Object.freeze`), and reused on every subsequent call. Depending on the selected type, the singleton can be component-scoped, global, or local.

To ensure unique identifiers, a combination of an internal global code and the `getElementId()` function is used, guaranteeing no collisions even when used across different modules.



## Signature

```typescript
function executeUse<R, O extends any[], RI extends ExecuteUseReturn<R>>(
  callback: (...args: O) => R,
  type: ExecuteUseType = ExecuteUseType.provide
): ((...args: O) => RI) | (() => RI)

function executeUseGlobal<R>(callback: () => R): () => ExecuteUseReturn<R>
function executeUseProvide<R, O extends any[]>(callback: (...args: O) => R): (() => ExecuteUseReturn<R>) | ((...args: O) => ExecuteUseReturn<R>)
function executeUseLocal<R, O extends any[]>(callback: (...args: O) => R): () => ExecuteUseReturn<R>
```

**Parameters:**
- `callback: (...args: O) => R` — The initializer function. Called on the first access and returns an object that will be frozen and cached. Arguments are passed only on the first call.
- `type: ExecuteUseType` — The type of singleton (default: `ExecuteUseType.provide`).

**Returns:** `ExecuteUseReturn<R>` — a frozen object extending `R` with the following methods:
- `init(): Readonly<R>` — returns the raw result of `callback` without any management methods (`init`, `destroyExecute`).
- `destroyExecute?(): void` — only on `local` and `global`. Resets the cache, allowing the object to be re-created on the next call.

## The `ExecuteUseType` Enum

### `provide` (default)

Ideal for state that should be shared within a component subtree. The first component that calls the function becomes the "owner": it performs `provide(id, instance)`. All nested components receive the same instance via `inject(id)`. If no injected value is found in the current component, a new instance is created and `provide` is called. Component-scoped singletons do not have a `destroyExecute()` method — their lifecycle is managed by Vue.

```typescript
import { executeUseProvide } from '@dxtmisha/functional'
import { ref } from 'vue'

const useTheme = executeUseProvide(() => {
  const mode = ref<'light' | 'dark'>('light')
  const toggle = () => { mode.value = mode.value === 'light' ? 'dark' : 'light' }
  return { mode, toggle }
})
```
// Parent component — creates provide:
const theme = useTheme()
theme.toggle()

// Child component — receives the same object via inject:
const theme = useTheme()

// Independent component outside the tree — a new instance is created:
const theme = useTheme()
```

### `global`

A single instance for the entire application. When using `executeUseGlobal`, the initializer is **not called immediately** — it is added to an internal queue. All global singletons are initialized when `executeUseGlobalInit()` is explicitly called, which is typically placed in `main.ts before mounting the app. Afterwards the queue is cleared, and all subsequent calls return the cached object.

```typescript
import { executeUseGlobal, executeUseGlobalInit } from '@dxtmisha/functional'
import { ref } from 'vue'

const useAuth = executeUseGlobal(() => {
  const user = ref<string | null>(null)
  const login = (name: string) => { user.value = name }
  const logout = () => { user.value = null }
  return { user, login, logout }
})
```
// In main.ts — before mounting the app:
executeUseGlobalInit()

// In any component:
const auth = useAuth()
auth.login('Alice')
auth.destroyExecute?.() // Force reset the cache when needed
```

### `local`

The cache lives in a closure. Does not use `provide/inject`. Suitable when you need a single instance without coupling to the Vue component tree. On the first call, the initializer is called with the provided arguments, the result is frozen and stored — on all subsequent calls, arguments are ignored and the same object is returned. The `destroyExecute()` method resets the cache, allowing the object to be re-created on the next call.

```typescript
import { executeUseLocal } from '@dxtmisha/functional'
import { ref } from 'vue'

const useCounter = executeUseLocal(() => {
  const count = ref(0)
  const increment = () => count.value++
  return { count, increment }
})
```
const counter = useCounter()
counter.increment()
console.log(counter.count.value) // 1

const same = useCounter() // same object
console.log(same === counter) // true

counter.destroyExecute?.()
const fresh = useCounter() // new object, count is 0 again
```

## `executeUseGlobalInit`

```typescript
function executeUseGlobalInit(): void
```

Calls all registered global initializers (`ExecuteUseType.global`) and clears the queue. Calling it again does not re-initialize already created singletons. Should be called **once** at app startup — before mounting the root component.

```typescript
import { createApp } from 'vue'
import { executeUseGlobalInit } from '@dxtmisha/functional'
import App from './App.vue'

executeUseGlobalInit()
createApp(App).mount('#app')
```
