# `executeUse`

`executeUse` là một factory bao bọc bất kỳ hàm khởi tạo nào và biến kết quả của nó thành một **singleton được quản lý**. Đối tượng được tạo đúng một lần, đóng băng (`Object.freeze`) và tái sử dụng trong mỗi lần gọi tiếp theo. Tùy theo loại được chọn, singleton có thể là phạm vi component, toàn cục, hoặc cục bộ.

Để đảm bảo định danh duy nhất, thư viện kết hợp mã code toàn cục nội bộ và hàm `getElementId()`, giúp ngăn chặn hoàn toàn việc trùng lặp ID ngay cả khi sử dụng trong nhiều module khác nhau.



## Chữ ký hàm

```typescript
function executeUse<R, O extends any[], RI extends ExecuteUseReturn<R>>(
  callback: (...args: O) => R,
  type: ExecuteUseType = ExecuteUseType.provide
): ((...args: O) => RI) | (() => RI)

function executeUseGlobal<R>(callback: () => R): () => ExecuteUseReturn<R>
function executeUseProvide<R, O extends any[]>(callback: (...args: O) => R): (() => ExecuteUseReturn<R>) | ((...args: O) => ExecuteUseReturn<R>)
function executeUseLocal<R, O extends any[]>(callback: (...args: O) => R): () => ExecuteUseReturn<R>
```

**Tham số:**
- `callback: (...args: O) => R` — Hàm khởi tạo. Được gọi khi truy cập lần đầu và trả về đối tượng sẽ được đóng băng và cache. Tham số chỉ được truyền vào ở lần gọi đầu tiên.
- `type: ExecuteUseType` — Loại singleton (mặc định: `ExecuteUseType.provide`).

**Trả về:** `ExecuteUseReturn<R>` — đối tượng đóng băng mở rộng `R` với các phương thức:
- `init(): Readonly<R>` — trả về kết quả gốc của `callback` mà không có các phương thức quản lý (`init`, `destroyExecute`).
- `destroyExecute?(): void` — chỉ có ở `local` và `global`. Đặt lại cache, cho phép tạo lại đối tượng ở lần gọi tiếp theo.

## Enum `ExecuteUseType`

### `provide` (mặc định)

Lý tưởng cho trạng thái cần được chia sẻ trong cây component. Component đầu tiên gọi hàm trở thành "chủ sở hữu": nó thực hiện `provide(id, instance)`. Tất cả component con nhận cùng instance qua `inject(id)`. Nếu không tìm thấy giá trị inject trong component hiện tại, một instance mới được tạo và `provide` được gọi. Singleton phạm vi component không có phương thức `destroyExecute()` — vòng đời của chúng được quản lý bởi Vue.

```typescript
import { executeUseProvide } from '@dxtmisha/functional'
import { ref } from 'vue'

const useTheme = executeUseProvide(() => {
  const mode = ref<'light' | 'dark'>('light')
  const toggle = () => { mode.value = mode.value === 'light' ? 'dark' : 'light' }
  return { mode, toggle }
})
```
// Component cha — tạo provide:
const theme = useTheme()
theme.toggle()

// Component con — nhận cùng đối tượng qua inject:
const theme = useTheme()

// Component độc lập ngoài cây — một instance mới được tạo:
const theme = useTheme()
```

### `global`

Một instance cho toàn bộ ứng dụng. Khi sử dụng `executeUseGlobal`, hàm khởi tạo **không được gọi ngay** — nó được thêm vào hàng đợi nội bộ. Tất cả singleton toàn cục được khởi tạo khi `executeUseGlobalInit()` được gọi tường minh, thường được đặt trong `main.ts` trước khi mount ứng dụng. Sau đó hàng đợi được xóa và mọi lần gọi tiếp theo đều trả về đối tượng đã cache.

```typescript
import { executeUseGlobal, executeUseGlobalInit } from '@dxtmisha/functional'
import { ref } from 'vue'

const useAuth = executeUseGlobal(() => {
  const user = ref<string | null>(null)
  const login = (name: string) => { user.value = name }
  const logout = () => { user.value = null }
  return { user, login, logout }
})
```
// Trong main.ts — trước khi mount ứng dụng:
executeUseGlobalInit()

// Trong bất kỳ component nào:
const auth = useAuth()
auth.login('Alice')
auth.destroyExecute?.() // Bắt buộc đặt lại cache khi cần
```

### `local`

Cache tồn tại trong một closure. Không dùng `provide/inject`. Phù hợp khi cần một instance duy nhất nhưng không muốn kết nối với cây component Vue. Ở lần gọi đầu tiên, hàm khởi tạo được gọi với các tham số đã truyền, kết quả được đóng băng và lưu trữ — ở tất cả các lần gọi sau, tham số bị bỏ qua và cùng đối tượng được trả về. Phương thức `destroyExecute()` đặt lại cache, cho phép tạo lại đối tượng ở lần gọi tiếp theo.

```typescript
import { executeUseLocal } from '@dxtmisha/functional'
import { ref } from 'vue'

const useCounter = executeUseLocal(() => {
  const count = ref(0)
  const increment = () => count.value++
  return { count, increment }
})
```
const counter = useCounter()
counter.increment()
console.log(counter.count.value) // 1

const same = useCounter() // cùng đối tượng
console.log(same === counter) // true

counter.destroyExecute?.()
const fresh = useCounter() // đối tượng mới, count trở về 0
```

## `executeUseGlobalInit`

```typescript
function executeUseGlobalInit(): void
```

Gọi tất cả các hàm khởi tạo toàn cục đã đăng ký (`ExecuteUseType.global`) và xóa hàng đợi. Gọi lại lần nữa sẽ không khởi tạo lại các singleton đã tồn tại. Cần được gọi **một lần** khi khởi động ứng dụng — trước khi mount component gốc.

```typescript
import { createApp } from 'vue'
import { executeUseGlobalInit } from '@dxtmisha/functional'
import App from './App.vue'

executeUseGlobalInit()
createApp(App).mount('#app')
```
