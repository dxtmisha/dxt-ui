# `executeUse`

`executeUse` — это фабрика, которая оборачивает любую функцию-инициализатор и превращает её результат в **управляемый синглтон**. Объект создаётся ровно один раз, замораживается (`Object.freeze`) и переиспользуется при каждом последующем вызове. В зависимости от выбранного типа синглтон может быть компонентным, глобальным или локальным.

Для обеспечения уникальности идентификаторов используется сочетание внутреннего глобального кода и функции `getElementId()`, что гарантирует отсутствие коллизий даже при множественном использовании в разных модулях.



## Сигнатура

```typescript
function executeUse<R, O extends any[], RI extends ExecuteUseReturn<R>>(
  callback: (...args: O) => R,
  type: ExecuteUseType = ExecuteUseType.provide
): ((...args: O) => RI) | (() => RI)

function executeUseGlobal<R>(callback: () => R): () => ExecuteUseReturn<R>
function executeUseProvide<R, O extends any[]>(callback: (...args: O) => R): (() => ExecuteUseReturn<R>) | ((...args: O) => ExecuteUseReturn<R>)
function executeUseLocal<R, O extends any[]>(callback: (...args: O) => R): () => ExecuteUseReturn<R>
```

**Параметры:**
- `callback: (...args: O) => R` — Функция-инициализатор. Вызывается при первом обращении и возвращает объект, который будет заморожен и закеширован. Аргументы передаются только при первом вызове.
- `type: ExecuteUseType` — Тип синглтона (по умолчанию `ExecuteUseType.provide`).

**Возвращает:** `ExecuteUseReturn<R>` — замороженный объект, расширяющий `R` следующими методами:
- `init(): Readonly<R>` — возвращает исходный результат `callback` без методов управления (`init`, `destroyExecute`).
- `destroyExecute?(): void` — только у `local` и `global`. Сбрасывает кеш, позволяя пересоздать объект при следующем вызове.

## Тип `ExecuteUseType`

### `provide` (по умолчанию)

Идеален для состояния, которое должно быть общим для поддерева компонентов. Первый компонент, вызвавший функцию, становится «владельцем»: он выполняет `provide(id, instance)`. Все вложенные компоненты получат тот же экземпляр через `inject(id)`. Если в текущем компоненте нет инжектированного значения — создаётся новый экземпляр и выполняется `provide`. Метод `destroyExecute()` у компонентных синглтонов отсутствует — жизненным циклом управляет сам Vue.

```typescript
import { executeUseProvide } from '@dxtmisha/functional'
import { ref } from 'vue'

const useTheme = executeUseProvide(() => {
  const mode = ref<'light' | 'dark'>('light')
  const toggle = () => { mode.value = mode.value === 'light' ? 'dark' : 'light' }
  return { mode, toggle }
})
```
// В родительском компоненте — создаётся provide:
const theme = useTheme()
theme.toggle()

// В дочернем компоненте — получает тот же объект через inject:
const theme = useTheme()

// В независимом компоненте вне дерева — создаётся новый экземпляр:
const theme = useTheme()
```

### `global`

Один экземпляр на всё приложение. При использовании `executeUseGlobal` инициализатор **не вызывается немедленно** — он добавляется во внутреннюю очередь. Инициализация всех глобальных синглтонов происходит при явном вызове `executeUseGlobalInit()`, который обычно размещается в `main.ts` до монтирования приложения. После этого очередь очищается и при любом последующем вызове метода возвращается закешированный объект.

```typescript
import { executeUseGlobal, executeUseGlobalInit } from '@dxtmisha/functional'
import { ref } from 'vue'

const useAuth = executeUseGlobal(() => {
  const user = ref<string | null>(null)
  const login = (name: string) => { user.value = name }
  const logout = () => { user.value = null }
  return { user, login, logout }
})
```
// В main.ts — до монтирования приложения:
executeUseGlobalInit()

// В любом компоненте:
const auth = useAuth()
auth.login('Alice')
auth.destroyExecute?.() // Принудительный сброс кеша при необходимости
```

### `local`

Кеш хранится в замыкании. Не использует `provide/inject`. Подходит, когда нужен единственный экземпляр без привязки к дереву компонентов Vue. При первом вызове инициализатор вызывается с переданными аргументами, результат замораживается и сохраняется — при всех последующих вызовах аргументы игнорируются и возвращается тот же объект. Метод `destroyExecute()` сбрасывает кеш, позволяя пересоздать объект при следующем вызове.

```typescript
import { executeUseLocal } from '@dxtmisha/functional'
import { ref } from 'vue'

const useCounter = executeUseLocal(() => {
  const count = ref(0)
  const increment = () => count.value++
  return { count, increment }
})
```
const counter = useCounter()
counter.increment()
console.log(counter.count.value) // 1

const same = useCounter() // тот же объект
console.log(same === counter) // true

counter.destroyExecute?.()
const fresh = useCounter() // новый объект, count снова 0
```

## `executeUseGlobalInit`

```typescript
function executeUseGlobalInit(): void
```

Вызывает все зарегистрированные глобальные инициализаторы (`ExecuteUseType.global`) и очищает очередь. Повторный вызов не пересоздаёт уже инициализированные синглтоны. Должна быть вызвана **один раз** при старте приложения — до монтирования корневого компонента.

```typescript
import { createApp } from 'vue'
import { executeUseGlobalInit } from '@dxtmisha/functional'
import App from './App.vue'

executeUseGlobalInit()
createApp(App).mount('#app')
```
