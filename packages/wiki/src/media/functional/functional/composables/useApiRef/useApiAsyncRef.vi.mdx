# `useApiAsyncRef`

`useApiAsyncRef` là một wrapper bất đồng bộ phía trên [`useApiRef`](./useApiRef.vi.mdx), thực hiện việc khởi tạo yêu cầu ngay lập tức.

Khác với `useApiRef` tiêu chuẩn, vốn được khởi tạo "lười" (chỉ khi truy cập dữ liệu lần đầu), phương thức này gọi `init()` ngay lập tức và trả về một `Promise` với đối tượng quản lý yêu cầu.

## Khi nào nên sử dụng
- Bên trong `async setup()` để đợi dữ liệu (`await`).
- Trong Server-Side Rendering (SSR) để đảm bảo có dữ liệu trước khi render.
- Khi yêu cầu cần được kích hoạt ngay khi gọi phương thức, không cần đợi truy cập vào các biến phản ứng.

## Tham số
Các tham số gần như hoàn toàn giống với `useApiRef`, ngoại trừ việc thêm vào `awaitFetch`.

- `path?: RefOrNormal<string | undefined>` — đường dẫn endpoint.
- `options?: ApiOptions` — tùy chọn yêu cầu.
- `awaitFetch?: boolean` — nếu là `true` (mặc định là `false` trong mã nguồn, nhưng khuyến nghị để `true` khi dùng `await`), phương thức sẽ đợi yêu cầu đầu tiên hoàn thành trước khi trả về kết quả.
- `reactivity?: boolean` — có bật tính phản ứng hay không.
- `conditions?: RefType<boolean>` — điều kiện thực thi.
- `transformation?: ...` — chuyển đổi dữ liệu.
- `validateResponseContract?: ...` — xác thực hợp đồng dữ liệu.
- `unmounted?: boolean` — có xóa khỏi cache khi unmount hay không.
- `apiInstance?: ApiInstance` — instance của API.

## Giá trị trả về
Trả về `Promise<UseApiRef<R>>`. Cấu trúc của đối tượng `UseApiRef` hoàn toàn giống với giá trị trả về của [`useApiRef`](./useApiRef.vi.mdx).

## Ví dụ

```typescript
import { useApiAsyncRef } from '@dxtmisha/functional'

// Bên trong async setup
const { data, loading } = await useApiAsyncRef(
  'users/list',
  { method: 'GET' },
  true // Đợi yêu cầu hoàn thành
)

// Tại thời điểm này, dữ liệu đã được tải xong (nếu awaitFetch: true)
console.log(data.value)
```
