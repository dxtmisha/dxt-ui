# `useApiRef`

Composable để làm việc với các yêu cầu API trong Vue component. Trả về dữ liệu phản ứng, cờ trạng thái và các phương thức quản lý tải. Sử dụng **lazy loading** — yêu cầu chỉ được gửi khi lần đầu truy cập vào thuộc tính `data`.

**Tham số:**
- `path?: RefOrNormal<string | undefined>` — đường dẫn endpoint (có thể là `Ref` phản ứng).
- `options?: ApiOptions` — tùy chọn yêu cầu (đối tượng `ApiFetch` hoặc chuỗi tên phương thức).
- `reactivity?: boolean` — tự động chạy lại yêu cầu khi `path`, `options` hoặc ngôn ngữ thay đổi. Mặc định: `true`.
- `conditions?: RefType<boolean>` — điều kiện thực thi phản ứng. Nếu `false` — yêu cầu không được thực hiện và dữ liệu bị xóa.
- `transformation?: (data: T) => ApiData<R>` — hàm chuyển đổi phản hồi trước khi lưu vào `data`.
- `unmounted?: boolean` — nếu `true`, dữ liệu bị xóa và watcher dừng lại khi component unmount.

**Trả về (`UseApiRef<R>`):**

Trạng thái phản ứng (getter `ComputedRef`):
- `data` — `Ref<ApiData<R> | undefined>` — dữ liệu đã tải. Kích hoạt yêu cầu (lazy) khi truy cập lần đầu.
- `starting` — `ComputedRef<boolean>` — `true` khi dữ liệu chưa bao giờ được tải (`data === undefined`).
- `loading` — `ComputedRef<boolean>` — `true` khi đang có yêu cầu tích cực đến máy chủ.
- `reading` — `ComputedRef<boolean>` — `true` khi đang trong quá trình đọc dữ liệu.

Phương thức:
- `isStarting(): boolean` — trả về giá trị hiện tại của cờ `starting`.
- `isLoading(): boolean` — trả về giá trị hiện tại của cờ `loading`.
- `isReading(): boolean` — trả về giá trị hiện tại của cờ `reading`.
- `getItem(): ApiData<R> | undefined` — trả về dữ liệu đã tải hiện tại mà không có tính phản ứng.
- `reset(): Promise<void>` — chạy lại yêu cầu một cách bắt buộc.
- `stop(): void` — dừng watcher và đặt lại `data` về `undefined`.
- `abort(): void` — hủy yêu cầu hiện tại qua `AbortController` mà không xóa dữ liệu.

## Lazy Loading

Yêu cầu **không được gửi** khi gọi `useApiRef`. Nó chỉ được kích hoạt khi lần đầu truy cập vào thuộc tính `data`. Điều này cho phép gọi composable ở đầu `setup()` mà không có các yêu cầu mạng không cần thiết.

## Điều kiện toàn cục

`setApiRefGlobalConditions(conditions)` — thiết lập điều kiện toàn cục cho tất cả các instance `useApiRef`. Chỉ có hiệu lực một lần.

## Ví dụ

```typescript
import { ref } from 'vue'
import { useApiRef, setApiRefGlobalConditions } from '@dxtmisha/functional'

const userId = ref('123')
const isAuthenticated = ref(true)

// Điều kiện toàn cục (tất cả yêu cầu chỉ chạy nếu đã xác thực)
setApiRefGlobalConditions(isAuthenticated)

const {
  data: user,
  starting,
  loading,
  reading,
  getItem,
  reset,
  stop,
  abort
} = useApiRef(
  () => `users/${userId.value}`,
  { method: 'GET' },
  true // Tự động chạy lại khi userId thay đổi
)

// Kiểm tra trạng thái không có tính phản ứng
console.log(getItem()) // undefined trước khi truy cập data lần đầu

// Tải lại bắt buộc
await reset()

// Hủy yêu cầu (không xóa dữ liệu)
abort()

// Dừng theo dõi (xóa dữ liệu)
stop()
```
