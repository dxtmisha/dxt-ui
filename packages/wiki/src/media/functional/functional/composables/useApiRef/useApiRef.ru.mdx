# `useApiRef`

Composable для удобной работы с API-запросами во Vue-компонентах. Возвращает реактивные данные, флаги состояния и методы управления загрузкой. Использует **ленивую загрузку** — запрос отправляется только при первом обращении к свойству `data`.

**Параметры:**
- `path?: RefOrNormal<string | undefined>` — путь к endpoint (может быть реактивным `Ref`).
- `options?: ApiOptions` — опции запроса (объект `ApiFetch` или строка с именем метода).
- `reactivity?: boolean` — автоматически перезапускать запрос при изменении `path`, `options` или локали. По умолчанию: `true`.
- `conditions?: RefType<boolean>` — реактивное условие выполнения. Если `false` — запрос не выполняется, данные сбрасываются.
- `transformation?: (data: T) => ApiData<R>` — функция преобразования ответа перед сохранением в `data`.
- `unmounted?: boolean` — если `true`, данные сбрасываются и watcher останавливается при размонтировании компонента.

**Возвращает (`UseApiRef<R>`):**

Реактивные состояния (геттеры `ComputedRef`):
- `data` — `Ref<ApiData<R> | undefined>` — загруженные данные. При первом обращении запускает запрос (lazy loading).
- `starting` — `ComputedRef<boolean>` — `true` пока данные еще ни разу не загружены (`data === undefined`).
- `loading` — `ComputedRef<boolean>` — `true` пока идет активный запрос к серверу.
- `reading` — `ComputedRef<boolean>` — `true` пока продолжается чтение данных.

Методы:
- `isStarting(): boolean` — возвращает значение флага `starting`.
- `isLoading(): boolean` — возвращает значение флага `loading`.
- `isReading(): boolean` — возвращает значение флага `reading`.
- `getItem(): ApiData<R> | undefined` — возвращает текущие загруженные данные без реактивности.
- `reset(): Promise<void>` — принудительно перезапускает запрос.
- `stop(): void` — останавливает watcher и сбрасывает `data` в `undefined`.
- `abort(): void` — отменяет текущий запрос через `AbortController` без сброса данных.

## Ленивая загрузка (Lazy Loading)

Запрос **не отправляется** при вызове `useApiRef`. Он запускается только при первом обращении к свойству `data`. Это позволяет вызывать composable наверху `setup()` без лишних сетевых запросов.

## Глобальные условия

`setApiRefGlobalConditions(conditions)` — устанавливает глобальное условие для всех экземпляров `useApiRef`. Работает только один раз.

## Пример

```typescript
import { ref } from 'vue'
import { useApiRef, setApiRefGlobalConditions } from '@dxtmisha/functional'

const userId = ref('123')
const isAuthenticated = ref(true)

// Глобальное условие (все запросы выполняются только если аутентифицированы)
setApiRefGlobalConditions(isAuthenticated)

const {
  data: user,
  starting,
  loading,
  reading,
  getItem,
  reset,
  stop,
  abort
} = useApiRef(
  () => `users/${userId.value}`,
  { method: 'GET' },
  true // Реактивно перезапускать при изменении userId
)

// Проверка состояния без реактивности
console.log(getItem()) // undefined до первого обращения к data

// Принудительная перезагрузка
await reset()

// Отмена запроса (без сброса данных)
abort()

// Остановка слежения (сбрасывает данные)
stop()
```
