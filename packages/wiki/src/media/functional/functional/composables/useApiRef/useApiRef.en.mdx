# `useApiRef`

A composable for working with API requests in Vue components. Returns reactive data, state flags, and load management methods. Uses **lazy loading** ‚Äî the request is only sent on the first access to the `data` property.

**Parameters:**
- `path?: RefOrNormal<string | undefined>` ‚Äî endpoint path (can be a reactive `Ref`).
- `options?: ApiOptions` ‚Äî request options (`ApiFetch` object or method name string).
- `reactivity?: boolean` ‚Äî automatically re-run the request when `path`, `options`, or locale change. Default: `true`.
- `conditions?: RefType<boolean>` ‚Äî reactive execution condition. If `false` ‚Äî the request is not made and data is cleared.
- `transformation?: (data: T) => ApiData<R>` ‚Äî response transformation function applied before saving to `data`.
- `unmounted?: boolean` ‚Äî if `true`, data is cleared and the watcher is stopped when the component unmounts.

**Returns (`UseApiRef<R>`):**

Reactive states (getter `ComputedRef`s):
- `data: Ref<ApiData<R> | undefined>` ‚Äî A reference to the loaded data. Implements **lazy loading**: the server request is automatically initiated on the first access to this property.
- `starting: ComputedRef<boolean>` ‚Äî Returns `true` if data has never been requested yet or is in the process of fetching the first portion of data (`data` is still `undefined`).
- `loading: ComputedRef<boolean>` ‚Äî Returns `true` if any network request (initial or subsequent `reset`) is currently in progress.
- `reading: ComputedRef<boolean>` ‚Äî A flag for an active data reading/processing process.

Methods:
- `isStarting(): boolean` ‚Äî A synchronous method that returns the current value of the `starting` flag.
- `isLoading(): boolean` ‚Äî A synchronous method that returns the current value of the `loading` flag.
- `isReading(): boolean` ‚Äî A synchronous method that returns the current value of the `reading` flag.
- `getItem(): ApiData<R> | undefined` ‚Äî Method to get the current value of `data` without creating a reactive dependency.
- `reset(): Promise<void>` ‚Äî An asynchronous method to force re-run the request. Cancels the current active request (if any) and initiates a new one.
- `stop(): void` ‚Äî Stops watching reactive dependencies and resets `data` to `undefined`.
- `abort(): void` ‚Äî Cancels the current HTTP request via the internal `AbortController`, while keeping already loaded data in `data`.

## Lazy Loading

The request is **not sent** when `useApiRef` is called. It is triggered only on the first access to the `data` property. This allows calling the composable at the top of `setup()` without unnecessary network requests.

## Global Conditions

`setApiRefGlobalConditions(conditions)` ‚Äî sets a global condition for all `useApiRef` instances. Only works on the first call.

## Example

```typescript
import { ref } from 'vue'
import { useApiRef, setApiRefGlobalConditions } from '@dxtmisha/functional'

const userId = ref('123')
const isAuthenticated = ref(true)

// Global condition (all requests only run if authenticated)
setApiRefGlobalConditions(isAuthenticated)

const {
  data: user,
  starting,
  loading,
  reading,
  getItem,
  reset,
  stop,
  abort
} = useApiRef(
  () => `users/${userId.value}`,
  { method: 'GET' },
  true // Reactively re-run on userId change
)

// Check state without reactivity
console.log(getItem()) // undefined before first access to data

// Force reload
await reset()

// Cancel request (without clearing data)
abort()

// Stop watching (clears data)
stop()
```
