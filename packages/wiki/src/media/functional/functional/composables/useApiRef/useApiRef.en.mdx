# `useApiRef`

A composable for working with API requests in Vue components. Returns reactive data, state flags, and load management methods. Uses **lazy loading** — the request is only sent on the first access to the `data` property.

**Parameters:**
- `path?: RefOrNormal<string | undefined>` — endpoint path (can be a reactive `Ref`).
- `options?: ApiOptions` — request options (`ApiFetch` object or method name string).
- `reactivity?: boolean` — automatically re-run the request when `path`, `options`, or locale change. Default: `true`.
- `conditions?: RefType<boolean>` — reactive execution condition. If `false` — the request is not made and data is cleared.
- `transformation?: (data: T) => ApiData<R>` — response transformation function applied before saving to `data`.
- `unmounted?: boolean` — if `true`, data is cleared and the watcher is stopped when the component unmounts.

**Returns (`UseApiRef<R>`):**

Reactive states (getter `ComputedRef`s):
- `data` — `Ref<ApiData<R> | undefined>` — loaded data. Triggers the request (lazy) on first access.
- `starting` — `ComputedRef<boolean>` — `true` while data has never been loaded yet (`data === undefined`).
- `loading` — `ComputedRef<boolean>` — `true` while an active request to the server is in progress.
- `reading` — `ComputedRef<boolean>` — `true` while data reading is in progress.

Methods:
- `isStarting(): boolean` — returns the current value of the `starting` flag.
- `isLoading(): boolean` — returns the current value of the `loading` flag.
- `isReading(): boolean` — returns the current value of the `reading` flag.
- `getItem(): ApiData<R> | undefined` — returns the current loaded data without reactivity.
- `reset(): Promise<void>` — force re-runs the request.
- `stop(): void` — stops the watcher and resets `data` to `undefined`.
- `abort(): void` — cancels the current request via `AbortController` without clearing data.

## Lazy Loading

The request is **not sent** when `useApiRef` is called. It is triggered only on the first access to the `data` property. This allows calling the composable at the top of `setup()` without unnecessary network requests.

## Global Conditions

`setApiRefGlobalConditions(conditions)` — sets a global condition for all `useApiRef` instances. Only works on the first call.

## Example

```typescript
import { ref } from 'vue'
import { useApiRef, setApiRefGlobalConditions } from '@dxtmisha/functional'

const userId = ref('123')
const isAuthenticated = ref(true)

// Global condition (all requests only run if authenticated)
setApiRefGlobalConditions(isAuthenticated)

const {
  data: user,
  starting,
  loading,
  reading,
  getItem,
  reset,
  stop,
  abort
} = useApiRef(
  () => `users/${userId.value}`,
  { method: 'GET' },
  true // Reactively re-run on userId change
)

// Check state without reactivity
console.log(getItem()) // undefined before first access to data

// Force reload
await reset()

// Cancel request (without clearing data)
abort()

// Stop watching (clears data)
stop()
```
