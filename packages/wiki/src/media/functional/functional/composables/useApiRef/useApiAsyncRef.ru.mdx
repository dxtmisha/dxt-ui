# `useApiAsyncRef`

`useApiAsyncRef` — это асинхронная обертка над [`useApiRef`](./useApiRef.ru.mdx), которая выполняет немедленную инициализацию запроса.

В отличие от стандартного `useApiRef`, который инициализируется "лениво" (при первом обращении к данным), этот метод вызывает `init()` сразу и возвращает `Promise` с объектом управления запросом.

## Когда использовать
- Внутри `async setup()` для ожидания данных (`await`).
- В серверном рендеринге (SSR) для гарантии получения данных до рендеринга.
- Когда требуется запустить запрос немедленно после вызова метода, не дожидаясь обращения к реактивным переменным.

## Параметры
Параметры почти идентичны `useApiRef`, за исключением добавленного `awaitFetch`.

- `path?: RefOrNormal<string | undefined>` — путь к endpoint.
- `options?: ApiOptions` — опции запроса.
- `awaitFetch?: boolean` — если `true` (по умолчанию `false` в коде, но рекомендуется `true` при использовании `await`), метод дождется окончания первого запроса перед тем, как вернуть результат.
- `reactivity?: boolean` — включить ли реактивность.
- `conditions?: RefType<boolean>` — условия выполнения.
- `transformation?: ...` — преобразование данных.
- `validateResponseContract?: ...` — валидация контракта.
- `unmounted?: boolean` — удалять ли из кэша при размонтировании.
- `apiInstance?: ApiInstance` — экземпляр API.

## Возвращаемое значение
Возвращает `Promise<UseApiRef<R>>`. Состав объекта `UseApiRef` полностью идентичен возвращаемому значению [`useApiRef`](./useApiRef.ru.mdx).

## Пример

```typescript
import { useApiAsyncRef } from '@dxtmisha/functional'

// Внутри async setup
const { data, loading, error } = await useApiAsyncRef(
  'users/list',
  { method: 'GET' },
  true // Ждем завершения запроса
)

// К этому моменту данные уже загружены (если awaitFetch: true)
console.log(data.value)
```
