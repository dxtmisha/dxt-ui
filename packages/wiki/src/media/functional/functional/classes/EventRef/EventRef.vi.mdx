# Lớp EventRef

Trình quản lý trình nghe sự kiện có tính phản ứng, giúp đơn giản hóa việc gắn và gỡ bỏ các sự kiện cho các thành phần DOM hoặc đối tượng `window`. Nó tự động cập nhật các đăng ký khi thành phần mục tiêu (được cung cấp dưới dạng Vue `Ref`) thay đổi.

Bao gồm việc tự động liên kết lại các trình nghe trong quá trình thay đổi thành phần động và tích hợp hoàn toàn với vòng đời của Vue.

## Các tính năng chính

- **Tính phản ứng** — tự động cập nhật khi thành phần `Ref` thay đổi.
- **Hỗ trợ đa loại** — nghe nhiều loại sự kiện cùng một lúc.
- **Tự động dọn dẹp** — quản lý bộ nhớ hiệu quả thông qua hệ thống phản ứng.

## Khởi tạo

Tạo một thực thể `EventRef` mới. Lớp này tự động đồng bộ hóa các trình lắng nghe sự kiện với trạng thái của Vue `ref` được truyền vào. Nếu phần tử đích thay đổi, `EventRef` sẽ tự động liên kết lại sự kiện.

### Ràng buộc Generic (Generic Constraints)

- `E extends ElementOrWindow` — kiểu của phần tử đích (ví dụ: `HTMLElement`, `Window`, hoặc `Document`).
- `O extends Event` — kiểu của đối tượng sự kiện (ví dụ: `MouseEvent`, `KeyboardEvent`).
- `D extends Record<string, any>` — kiểu dữ liệu tùy chỉnh (`detail`) gắn liền với sự kiện.

**Tham số:**
- `elementSelector?: RefOrNormal<ElementOrString<E> | undefined>` — phần tử đích hoặc bộ chọn (có thể ở dạng phản ứng).
- `elementSelectorControl?: RefOrNormal<ElementOrString<HTMLElement>>` — phần tử điều khiển. Nếu phần tử này bị xóa khỏi DOM, việc lắng nghe sẽ dừng lại.
- `type: string | string[] = ['click']` — loại sự kiện hoặc mảng các loại sự kiện.
- `listener?: EventListenerDetail<O, D>` — hàm xử lý.
- `options?: EventOptions` — các tùy chọn trình lắng nghe (`capture`, `passive`, `once`).
- `detail?: D` — dữ liệu bổ sung của sự kiện.

```typescript
import { EventRef } from '@dxtmisha/functional'
import { ref, onMounted, onUnmounted } from 'vue'

// 1. Khởi tạo đơn giản với Ref
const buttonRef = ref<HTMLButtonElement>()
const event = new EventRef(buttonRef, undefined, 'click', (e) => {
  console.log('Nút đã được nhấn:', e.target)
})

// 2. Sử dụng nhiều sự kiện và tùy chọn
const scrollEvent = new EventRef(
  window, 
  undefined, 
  ['scroll', 'resize'], 
  () => console.log('Cửa sổ đã thay đổi'),
  { passive: true }
)

onMounted(() => {
  event.start()
  scrollEvent.start()
})

onUnmounted(() => {
  event.stop()
  scrollEvent.stop()
})
```

## Vòng đời & Điều khiển

- `start(): this` — Kích hoạt việc nghe sự kiện trên thành phần mục tiêu.
- `stop(): this` — Dừng nghe và gỡ bỏ tất cả các trình nghe.
- `toggle(activity: boolean): this` — Bật hoặc tắt việc nghe sự kiện dựa trên cờ flag.
- `reset(): this` — Tải lại các trình nghe sự kiện (hữu ích khi thay đổi cài đặt).

## Trạng thái & Truy xuất

- `isActive(): boolean` — Kiểm tra xem việc nghe có đang hoạt động hay không.
- `getElement(): E | undefined` — Trả về thành phần mục tiêu hiện tại để nghe.

## Cấu hình

- `setElement(element: ElementOrString): this` — Thay đổi thành phần mục tiêu cho trình nghe sự kiện. Các trình nghe hiện tại sẽ tự động được chuyển sang đối tượng mới.
- `setElementControl(element: ElementOrString): this` — Thiết lập thành phần điều khiển để xác minh sự hiện diện trong DOM. Việc nghe sẽ tự động dừng nếu thành phần này bị xóa khỏi DOM.
- `setType(type: string | string[]): this` — Cập nhật các loại sự kiện để nghe (ví dụ: `click` hoặc `['mousedown', 'mouseup']`).
- `setListener(listener: Function): this` — Thay đổi hàm callback (trình xử lý).
- `setOptions(options: EventOptions): this` — Thay đổi các tùy chọn trình nghe (ví dụ: `once`, `passive`, `capture`).
- `setDetail(detail: object): this` — Thiết lập dữ liệu bổ sung có sẵn trong trình xử lý thông qua đối tượng sự kiện.

## Hành động

- `dispatch(detail?: object): this` — Kích hoạt sự kiện theo cách thủ công trên thành phần mục tiêu.
