# Класс DesignAbstract

Абстрактный базовый класс, предназначенный для управления свойствами конструктора и изменениями состояния. Он предоставляет структурированный способ обработки входных свойств, отслеживания изменений и выполнения обратных вызовов при обновлении данных.

## Ключевые особенности

- **Управление свойствами** — структурированная обработка входных данных (`props`) и хранилища событий (`event`).
- **Отслеживание изменений** — интеграция с `DesignChanged` для эффективного мониторинга и обновления изменений свойств.
- **Управление жизненным циклом** — автоматическое выполнение `makeCallback` и `initEvent` при обновлении данных.

## Инициализация

```typescript
import { DesignAbstract } from '@dxtmisha/functional'

interface MyProps { title: string }
interface MyEvent { title: string }

class MyDesign extends DesignAbstract<MyProps, MyEvent> {
  protected initEvent(): void {
    this.event.title = this.props.title
  }
}

const design = new MyDesign({ title: 'Привет' }, (event) => {
  console.log('Обновлено:', event)
})
```

## Управление данными

### `make`

Инициирует процесс обновления обратного вызова. Вызывает `makeCallback` и возвращает экземпляр для цепочки вызовов.

**Параметры:**
- `compelled?: boolean` — если `true`, принудительно обновляет данные независимо от наличия изменений.

**Возвращает:** `this`

### `makeCallback`

Основная логика выполнения для обновления состояния. Проверяет наличие изменений или принудительное обновление, затем инициализирует событие и запускает обратные вызовы.

**Параметры:**
- `compelled: boolean` — принудительно обновляет данные. По умолчанию: `false`.

**Возвращает:** `void`

## Внутреннее устройство

Внутри `DesignAbstract` использует класс `DesignChanged` для отслеживания состояния свойств. Это позволяет классу запускать обновления только тогда, когда данные действительно изменились, оптимизируя производительность сложных UI-компонентов.
