# RouterItemRef Class

A static utility class for managing Vue Router integration. It provides centralized access to the router instance, link resolution, and programmatic navigation, ensuring consistent routing behavior across the library components.

## Key Features

- **Static Interface** — no need to instantiate; all methods are available statically.
- **Link Resolution** — easily convert route names and parameters into URLs.
- **Unified Navigation** — provides a standard way to perform programmatic navigation.
- **Href Generation** — specialized methods to generate `ConstrHrefProps` for component links.

## Initialization & Usage

```typescript
import { RouterItemRef } from '@dxtmisha/functional'
import { useRouter } from 'vue-router'

// In your main entry or setup
const router = useRouter()
RouterItemRef.setOneTime(router)

// Getting a link
const url = RouterItemRef.getLink('profile', { id: 123 })
```

## Access and Configuration

- `get(): Router | undefined` — Returns the current router instance.
- `set(router: Router): void` — Sets or updates the global router instance.
- `setOneTime(router: Router): void` — Sets the router instance only if it hasn't been set yet.

## Link Resolution

- `getLink(name: string, params?, query?): string | undefined` — Resolves a route name and parameters into a full URL (href).
- `getHref(name?, params?, query?): ConstrHrefProps` — Returns an object containing the resolved `href` property.
- `rawToHref(to?): ConstrHrefProps` — Converts a raw route location object or string into `ConstrHrefProps`.

## Navigation

- `push(to: string | RouteLocationRaw): void` — Performs programmatic navigation to a new path.

## ConstrHrefProps Object

A standard object used by components for link properties:

- `href?: string` — the resolved URL for the link.
