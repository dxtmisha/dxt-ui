# DesignComp Class

A class for working with connected components and managing their rendering logic within a design system. It allows for component registration, modification of their default properties, and flexible VNode generation.

## Key Features

- **Component Management** — centralized registration and retrieval of Vue components.
- **Dynamic Modifications** — supports applying global or specific property modifications to components before rendering.
- **Flexible Rendering** — provides multiple methods (`render`, `renderOne`, `renderAdd`) for generating Vue VNodes with automatic property binding.
- **Performance Optimization** — uses internal caching with Vue `computed` properties for property modifications.

## Initialization

```typescript
import { DesignComp } from '@dxtmisha/functional'
import MyButton from './MyButton.vue'

const components = {
  button: MyButton
}

const modification = {
  button: { size: 'large' }
}

const design = new DesignComp(components, modification)
```

## Component Retrieval

- `is(name: string): boolean` — Checks if a specific component is registered.
- `get(name: string): COMP[K]` — Returns the registered component object by its name.
- `getModification(index?: string, props?: object): object | undefined` — Returns modified input data for connected components.

## Rendering Methods

- `render(name: string, props?: object, children?: any, index?: string): VNode[]` — Renders a component and returns an array containing the resulting VNode.
- `renderOne(name: string, props?: object, children?: any, index?: string): VNode | undefined` — Renders a single component and returns the VNode directly.
- `renderAdd(item: any[], name: string, props?: object, children?: any, index?: string): this` — Renders a component and pushes the result into a provided array.

## Internal Structure

`DesignComp` acts as a bridge between the design logic and the Vue rendering engine. It handles complex property merges via `getModification` and ensures that component modifications are reactive by leveraging Vue's `computed` system inside its inherited `DesignComponents` logic.
