# DataStorage Class

A versatile class for interacting with the browser's `localStorage` and `sessionStorage`. It features built-in prefixes, cache expiration, memory caching for active instances, and automatic JSON serialization.

## Key Features

- **Automatic Serialization** ‚Äî seamlessly handles JSON parsing and stringifying for stored configurations and objects.
- **Time-based Caching** ‚Äî allows saving objects with timestamps and respects age constraints during data retrieval.
- **Prefix Management** ‚Äî supports global and instance-specific prefixes to isolate and group keys within `localStorage`.
- **Memory Efficiency** ‚Äî creates singletons for identical keys, avoiding duplicate reads from disk.

## Initialization

To initialize the object, call the `DataStorage(name, isSession)` constructor.

**Parameters:**
- `name: string` ‚Äî the name of the storage key.
- `isSession: boolean = false` ‚Äî whether to use `sessionStorage` (defaults to `localStorage`).

```typescript
import { DataStorage } from '@dxtmisha/functional'

// 1. Create an instance for localStorage
const settings = new DataStorage('settings')

// 2. Create an instance for sessionStorage
const session = new DataStorage('user_session', true)
```

## Global Settings

- `setPrefix(newPrefix: string): void` (static) ‚Äî Changes the global prefix applied to new instances attempting to use `localStorage`. The default prefix is `ui-storage`.

## Data Management

### `get`

Retrieves the currently saved structured configuration.

**Parameters:**
- `defaultValue?: T | (() => T)` ‚Äî a fallback value to return if the storage is empty.
- `cache?: number` ‚Äî maximum allowed age (in seconds). If the data is older, it behaves as if nothing was stored.

**Returns:** `T | undefined` ‚Äî the stored value or `undefined`.

```javascript
// Get data, falling back to an empty object if not found
const currentSettings = settings.get({})

// Get data only if it was saved within the last 60 seconds
const cachedItem = settings.get(undefined, 60)
```

### `set`

Sets or updates the value in the storage mechanism.

**Parameters:**
- `value?: T | (() => T)` ‚Äî the value to save. Passing `undefined` will delete the item from storage.

**Returns:** `T | undefined` ‚Äî the stored value.

```javascript
settings.set({ theme: 'dark', notifications: true })
```

## Additional Methods

- `remove(): this` ‚Äî Clears the corresponding key actively from the `localStorage` or `sessionStorage` environments, and immediately unsets the existing memory instance.
- `update(): this` ‚Äî Forces the memory instance to synchronize tightly with the current raw data inside the physical Storage APIs.
