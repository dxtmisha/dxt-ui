# Класс UrlItem

Класс `UrlItem` представляет собой **изоморфный утилитарный класс** для парсинга, управления и обновления URL-адресов как на стороне клиента (DOM), так и на стороне сервера (SSR). Он инкапсулирует стандартный объект `URL`, предоставляя чистые геттеры свойств только для чтения и вспомогательные методы для работы с параметрами запроса.

## Ключевые особенности

- **Изоморфная поддержка** — автоматически определяет базовый URL-адрес в зависимости от среды выполнения (window.location в браузере, localhost при серверном рендеринге).
- **Изоляция запросов в SSR** — предоставляет метод `getInstance()`, поддерживаемый `ServerStorage`, для изоляции контекстов URL внутри каждого потока запроса.
- **Вспомогательные методы параметров** — предоставляет чистые методы с поддержкой цепочки вызовов (chaining) для проверки, получения, установки или удаления параметров запроса без ручного форматирования строки поиска.

## Методы

### Аксессоры

- `get href(): string` — Возвращает полную строку URL.
- `get protocol(): string` — Возвращает протокол, включая двоеточие на конце.
- `get username(): string` — Возвращает имя пользователя.
- `get password(): string` — Возвращает пароль.
- `get host(): string` — Возвращает хост (имя хоста и порт).
- `get hostname(): string` — Возвращает имя хоста (без порта).
- `get port(): string` — Возвращает порт.
- `get pathname(): string` — Возвращает путь к ресурсу.
- `get search(): string` — Возвращает строку запроса, включая ведущий вопросительный знак.
- `get searchParams(): URLSearchParams` — Возвращает стандартный объект `URLSearchParams`.
- `get hash(): string` — Возвращает хеш-строку, включая ведущий знак решетки.
- `get origin(): string` — Возвращает источник URL (только для чтения).
- `getParam(name: string): string | undefined` — Возвращает значение параметра запроса или `undefined`, если параметр не найден.
- `getParams(): Record<string, any>` — Возвращает все параметры запроса в виде объекта с преобразованными типами.

### Проверка

- `hasParam(name: string): boolean` — Проверяет, существует ли указанный параметр запроса.

### Действие

- `set(url?: string | URL): this` — Обновляет значение URL.
- `setParam(name: string, value: string): this` — Устанавливает значение параметра запроса.
- `setParams(params: Record<string, any>): this` — Заменяет все параметры запроса на указанный объект.
- `deleteParam(name: string): this` — Удаляет параметр запроса.

### Вспомогательные

- `static getInstance(): UrlItem` — Возвращает изолированный в рамках запроса экземпляр UrlItem.
- `toString(): string` — Преобразует экземпляр Url в его строковое представление.
- `toJSON(): string` — Преобразует экземпляр Url в строковое представление JSON.

## Архитектура в SSR

Класс `UrlItem` обеспечивает изолированное управление экземплярами в рамках запроса с помощью `ServerStorage`.

### Управление экземпляром

При вызове `UrlItem.getInstance()` класс извлекает или инициализирует экземпляр, хранящийся в `ServerStorage` под уникальным ключом (`__ui:url-item__`). Эта архитектура решает две основные проблемы:

1.  **Потокобезопасность в SSR**: Во время серверного рендеринга несколько серверных запросов выполняются одновременно. `ServerStorage` гарантирует, что каждый запрос имеет свой собственный изолированный контекст `UrlItem`, предотвращая утечку параметров запроса между сессиями пользователей.
2.  **Простота синглтона на клиенте**: На стороне клиента класс ведет себя как настоящий синглтон, позволяя компонентам глобально читать и обновлять параметры URL.
