# Класс Api

Статический класс-обертка над Fetch API для эффективной работы с сетевыми запросами. Предоставляет расширенные возможности конфигурирования, глобальной обработки статусов, эмуляции ответов, установки хуков подготовки и автоматического парсинга данных. 

Включает тесную интеграцию с зависимыми классами: `ApiStatus`, `ApiResponse`, `ApiPreparation`, `ApiDefault` и `ApiHeaders`.

## Ключевые особенности

- **CRUD методы** — удобные алиасы `get`, `post`, `put`, `delete` для работы с HTTP-запросами
- **Глобальная конфигурация** — установка базового URL, заголовков и параметров по умолчанию для всех запросов
- **Хуки жизненного цикла** — возможность задать функции, выполняемые до (`setPreparation`) и после (`setEnd`) каждого запроса
- **Отслеживание статуса** — доступ к статусу и ошибкам последнего запроса через `ApiStatus`
- **Кэширование и эмуляция** — управление мок-ответами и кэшированием через `ApiResponse`
- **Автоматический парсинг** — преобразование тела запроса в JSON/FormData и автоматическое извлечение данных из JSON-ответов
- **Геолокация** — автоматическая подстановка `{locale}`, `{country}`, `{language}` в пути запросов

## Методы получения инстансов

### `isLocalhost`

Определяет, запущен ли код на локальном сервере (возвращает `true`, если `location.hostname === 'localhost'`).

**Возвращает:** `boolean`

### `getStatus`

Возвращает менеджер статусов `ApiStatus` для работы со статусом последнего запроса, ошибками и ответами.

**Возвращает:** `ApiStatus`

### `getResponse`

Возвращает менеджер ответов/эмуляции `ApiResponse` для управления мок-ответами (полезно при разработке без backend API).

**Возвращает:** `ApiResponse`

## Эмуляция ответов (ApiResponse)

Менеджер эмуляции `ApiResponse` (доступен через `Api.getResponse()`) позволяет настраивать мок-ответы для определенных путей — это невероятно полезно при разработке UI без готового Backend API или для написания тестов.

### Регистрация мок-ответов

Метод `add` принимает объект типа `ApiResponseItem` или массив таких объектов.

**Важные поля `ApiResponseItem`:**
- `path` — Строка (точное совпадение) или Регулярное выражение (`RegExp`) для перехвата пути.
- `method` — HTTP метод (`ApiMethodItem.get`, `post` и т.д.).
- `request` — (Опционально) Фильтр по телу запроса. Можно использовать специальный маркер `'*any'`, чтобы перехватывать любые данные.
- `response` — Статический объект с данными ответа _или_ функция `(request) => any`, генерирующая ответ динамически на основе параметров запроса.
- `lag` — (Опционально) Имитировать задержку сети (`boolean`), добавляет случайный setTimeout от 0 до 2000мс.

```javascript
const responseManager = Api.getResponse()

// Включить режим разработчика (в консоль будут выводиться логи перехваченных запросов)
responseManager.setDevMode(true)

// Пример 1: Статический ответ на GET запрос
responseManager.add({
  path: 'users/profile', // можно использовать строку или RegExp
  method: 'GET',
  response: { id: 1, name: 'Admin', role: 'superuser' },
  lag: true // Искусственная задержка (lag) для симуляции реального сетевого пинга
})

// Пример 2: Динамический ответ с функцией (полезно для пагинации или поиска)
responseManager.add({
  path: /users\/search/, // перехватываем по RegExp
  method: 'GET',
  // Функция получает оригинальные параметры, переданные в request
  response: (request) => {
    return [
        { id: 1, name: `Искали: ${request.query}` }
    ]
  }
})

// Пример 3: Перехват POST запроса с любым телом
responseManager.add({
  path: 'users/create',
  method: 'POST',
  request: '*any', // маркер *any позволяет игнорировать проверку тела запроса
  response: { success: true, message: 'Пользователь успешно создан (MOCK)' }
})

// Использование в коде (сетевой запрос не уйдет, вернется мок)
const profile = await Api.get({ path: 'users/profile' })
```

### Приоритет и фильтрация моков

- Эмулятор (mock) срабатывает **до** реального `fetch`.
- Если эмулятор не находит совпадений (по `path`, `method` или `request`), класс `Api` выполнит реальный сетевой запрос к серверу.
- Вы можете отключить определенный мок, передав функцию в поле `disable: () => true`. Метод `add` не будет его перехватывать.

## Формирование запроса

### `getUrl`

Формирует полный адрес к скрипту запроса с автоматической подстановкой гео-меток: `{locale}`, `{country}`, `{language}`.

**Параметры:**
- `path: string` — путь к скрипту/эндпоинту.
- `api?: boolean` — (По умолчанию: `true`) Добавлять ли базовый путь API.

**Возвращает:** `string` — полный сформированный путь

### `getBody`

Формирует данные для тела (body) запроса (не применяется для `GET`). Может возвращать `FormData` или сериализованную JSON-строку.

**Параметры:**
- `request?: ApiFetch['request']` — данные запроса.
- `method?: string` — HTTP метод (по умолчанию `ApiMethodItem.get`).

**Возвращает:** `string | FormData | undefined`

### `getBodyForGet`

Формирует строку запроса (query-параметры) для подстановки в URL при использовании метода `GET`.

**Параметры:**
- `request: ApiFetch['request']` — данные запроса.
- `path?: string` — базовый путь запроса.
- `method?: string` — HTTP метод.

**Возвращает:** `string` — строка query параметров, начинающаяся с `?` или `&`

## Конфигурация

Все методы конфигурации возвращают класс `Api` для цепочки вызовов (chaining).

### `setHeaders`

Устанавливает глобальные заголовки по умолчанию (например, для авторизации), применяемые ко всем запросам.

**Параметры:**
- `headers: Record<string, string>` — список дополнительных заголовков.

**Возвращает:** `Api`

### `setRequestDefault`

Устанавливает общие параметры, которые будут автоматически добавляться ко всем последующим запросам (в тело или query параметры).

**Параметры:**
- `request: Record<string, any>` — список параметров по умолчанию.

**Возвращает:** `Api`

### `setUrl`

Изменяет базовый URL путь для API скриптов (по умолчанию \`/api/\`).

**Параметры:**
- `url: string` — базовый путь к API.

**Возвращает:** `Api`

### `setPreparation`

Устанавливает функцию-хук, которая автоматически выполняется _перед_ каждым сетевым запросом. Полезно для вывода глобального прелоадера или модификации конфигурации.

**Параметры:**
- `callback: (apiFetch: ApiFetch) => Promise<void>` — асинхронная функция, выполняемая перед запросом. Принимает конфигурацию текущего запроса.

**Возвращает:** `Api`

### `setEnd`

Устанавливает функцию-хук, выполняемую _после_ получения ответа от сервера, но до его парсинга. Если функция возвращает `{ reset: true }`, запрос повторяется.

**Параметры:**
- `callback: (query: Response, apiFetch: ApiFetch) => Promise<ApiPreparationEnd>` — обработчик ответа сервера. Получает сам ответ и исходные опции запроса.

**Возвращает:** `Api`

## Отмена запросов (AbortController)

API поддерживает отмену запросов из коробки. Для этого можно передать экземпляр `AbortController` в поле `controller` объекта `ApiFetch`.

```javascript
const controller = new AbortController()

// Запускаем запрос, передавая контроллер
Api.get({ path: 'users', controller }).catch(e => {
  if (e.name === 'AbortError') {
    console.log('Запрос был отменен пользователем')
  }
})

// Отменяем при размонтировании компонента (например)
controller.abort()
```

## Методы запросов

### `request`

Универсальный метод для выполнения сетевого HTTP-запроса. Если передана строка, выполняется простой `GET` запрос по этому адресу.
В случае сетевых сбоев (например, 500 статус) метод явно выбрасывает исключение (`throw e`), что позволяет перехватывать ошибки в `try/catch`. Если сервер возвращает JSON объект, к возвращаемым данным автоматически добавляется поле `statusObject` (инстанс статуса запроса).

**Параметры:**
- `pathRequest: string | ApiFetch` — путь (строка) или конфигурационный объект запроса.

**Возвращает:** `Promise<ApiData<T>>` — возвращает спарсенные данные ответа сервера (с полями `data`, `statusObject` и др., если возвращается объект).

### `get`

Отправляет запрос методом `GET`.

**Параметры:**
- `request: ApiFetch` — конфигурационный объект запроса (содержащий `path`, `request` и т.д.).

**Возвращает:** `Promise<T>`

### `post`

Отправляет запрос методом `POST`. Идеально для создания/сохранения сущностей.

**Параметры:**
- `request: ApiFetch` — конфигурационный объект запроса (содержащий `path`, `request` и т.д.).

**Возвращает:** `Promise<T>`

### `put`

Отправляет запрос методом `PUT`. Применяется для обновления данных.

**Параметры:**
- `request: ApiFetch` — конфигурационный объект запроса (содержащий `path`, `request` и т.д.).

**Возвращает:** `Promise<T>`

### `delete`

Отправляет запрос методом `DELETE`. Применяется для удаления данных из системы.

**Параметры:**
- `request: ApiFetch` — конфигурационный объект запроса (содержащий `path`, `request` и т.д.).

**Возвращает:** `Promise<T>`

## Примеры

```javascript
import { Api } from '@dxtmisha/functional'

// Конфигурация API
Api.setUrl('https://api.example.com/v1/')
   .setHeaders({ 'Authorization': 'Bearer token123' })
   .setRequestDefault({ source: 'webapp' })

// Отслеживание подготовки перед началом (hook)
Api.setPreparation(async (apiFetch) => {
    console.log('Начинается запрос к:', apiFetch.path)
})

// Простой GET запрос с обработкой ошибок
try {
  const data = await Api.get({ path: 'users' })
  // Из объекта ответа также можно извлечь встроенный статус (если возвращается объект):
  // console.log(data.statusObject.getStatus())
} catch (e) {
  console.error('Сетевая ошибка при запросе пользователей:', e)
}

// POST запрос с телом и возможностью отмены
const abortCtrl = new AbortController()
try {
  const newUser = await Api.post({
    path: 'users/create',
    request: { name: 'John Doe', age: 30 },
    controller: abortCtrl
  })
} catch (e) {
  if (e.name === 'AbortError') console.log('Запрос отменен!')
}

// Отмена в случае необходимости
// abortCtrl.abort()
```
