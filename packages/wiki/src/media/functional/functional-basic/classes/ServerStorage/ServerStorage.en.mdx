# ServerStorage Class

`ServerStorage` is a mechanism for managing **isolated global state** in applications with SSR (Server-Side Rendering) support.

The class allows you to work with data as easily as with static variables, but ensures that user data on the server does not intermingle between parallel requests.

## Key Features

1.  **Request Isolation**: Data on the server is bound to the context of the current HTTP request.
2.  **Hydration**: Ability to automatically transfer state from the server to the client via JSON.
3.  **Security**: Automatic XSS escaping during data transfer.

```typescript
import { ServerStorage } from '@dxtmisha/functional-basic'

// Save data with a hydration flag (will be available on the client)
ServerStorage.set('theme', () => 'dark', true)

// Retrieve data
const theme = ServerStorage.get('theme') // 'dark'
```

## Methods

### `init`
```typescript
static init(listener: () => Record<string, any>): typeof ServerStorage
```
Initializes the storage with a context listener. On the server, this listener should return an object unique to each request.

### `get`
```typescript
static get<T = any>(key: string, defaultValue?: () => T, hydration: boolean = false): T
```
Retrieves a value by key.
- `defaultValue`: if the key is not found, this function is called, the result is saved and returned.
- `hydration`: if `true`, the value will be included in the JSON for the client.

### `set`
```typescript
static set<T = any>(key: string, value: () => T, hydration: boolean = false): T
```
Saves a value to the storage.

### `has`
```typescript
static has(key: string): boolean
```
Checks for the existence of a key in the current context.

### `toString`
```typescript
static toString(): string
```
Generates an HTML `<script type="application/json">` tag containing all data marked for hydration. This method is called on the server within the page template.

## Isolation in SSR (Vue / Nuxt)

To work correctly on the server, you must pass the request context. In Vue 3.5+, this can be done via `useId` or a specific context object.

```typescript
import { useId } from 'vue'
import { ServerStorage } from '@dxtmisha/functional-basic'

// Initialization (e.g., in a plugin or root component)
ServerStorage.init(() => {
  // On the server, return an object bound to the request
  // On the client, this can be just a global object
  return typeof window === 'undefined' ? (globalContext || {}) : window
})
```

## Hydration (Transferring Data to Client)

To make data set on the server "come alive" in the browser:

1.  On the server, insert the result of `ServerStorage.toString()` into the HTML (usually before the closing `</body>`).
```html
<!-- Server output -->
<body>
  <div id="app">...</div>
  <script id="__ui:server:storage:id__" type="application/json">{"theme":"dark"}</script>
</body>
```

2.  In the browser, upon the first call to `get()` or `has()`, the class will automatically find this script and restore all values.

---

> **Important:** `ServerStorage` automatically escapes the `</script>` sequence within your data, preventing XSS attacks during hydration.
