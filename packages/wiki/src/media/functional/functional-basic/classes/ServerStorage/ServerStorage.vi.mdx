# Lớp ServerStorage

`ServerStorage` là một cơ chế để quản lý **trạng thái toàn cục được cách ly** trong các ứng dụng hỗ trợ SSR (Server-Side Rendering).

Lớp này cho phép bạn làm việc với dữ liệu dễ dàng như với các biến tĩnh (static), nhưng đảm bảo rằng dữ liệu người dùng trên máy chủ không bị trộn lẫn giữa các yêu cầu song song.

## Các tính năng chính

1.  **Cách ly yêu cầu**: Dữ liệu trên máy chủ được liên kết với ngữ cảnh của yêu cầu HTTP hiện tại.
2.  **Hydration**: Khả năng tự động chuyển trạng thái từ máy chủ sang máy khách thông qua JSON.
3.  **Bảo mật**: Tự động thoát (escape) XSS trong quá trình truyền dữ liệu.

```typescript
import { ServerStorage } from '@dxtmisha/functional-basic'

// Lưu dữ liệu với cờ hydration (sẽ có sẵn trên máy khách)
ServerStorage.set('theme', () => 'dark', true)

// Truy xuất dữ liệu
const theme = ServerStorage.get('theme') // 'dark'
```

## Phương thức

### `init`
```typescript
static init(listener: () => Record<string, any>): typeof ServerStorage
```
Khởi tạo kho lưu trữ với một bộ lắng nghe ngữ cảnh. Trên máy chủ, bộ lắng nghe này sẽ trả về một đối tượng duy nhất cho mỗi yêu cầu.

### `get`
```typescript
static get<T = any>(key: string, defaultValue?: () => T, hydration: boolean = false): T
```
Truy xuất giá trị theo khóa.
- `defaultValue`: nếu không tìm thấy khóa, hàm này sẽ được gọi, kết quả được lưu lại và trả về.
- `hydration`: nếu là `true`, giá trị sẽ được bao gồm trong JSON để gửi xuống máy khách.

### `set`
```typescript
static set<T = any>(key: string, value: () => T, hydration: boolean = false): T
```
Lưu một giá trị vào kho lưu trữ.

### `has`
```typescript
static has(key: string): boolean
```
Kiểm tra sự tồn tại của một khóa trong ngữ cảnh hiện tại.

### `toString`
```typescript
static toString(): string
```
Tạo một thẻ HTML `<script type="application/json">` chứa tất cả dữ liệu được đánh dấu để hydration. Phương thức này được gọi trên máy chủ trong mẫu trang (page template).

## Cách ly trong SSR (Vue / Nuxt)

Để hoạt động chính xác trên máy chủ, bạn phải truyền ngữ cảnh yêu cầu. Trong Vue 3.5+, việc này có thể được thực hiện thông qua `useId` hoặc một đối tượng ngữ cảnh cụ thể.

```typescript
import { useId } from 'vue'
import { ServerStorage } from '@dxtmisha/functional-basic'

// Khởi tạo (ví dụ: trong một plugin hoặc thành phần gốc)
ServerStorage.init(() => {
  // Trên máy chủ, trả về một đối tượng liên kết với yêu cầu
  // Trên máy khách, đây có thể chỉ là một đối tượng toàn cục
  return typeof window === 'undefined' ? (globalContext || {}) : window
})
```

## Hydration (Chuyển dữ liệu sang máy khách)

Để làm cho dữ liệu được thiết lập trên máy chủ "sống lại" trong trình duyệt:

1.  Trên máy chủ, chèn kết quả của `ServerStorage.toString()` vào HTML (thường là trước thẻ đóng `</body>`).
```html
<!-- Server output -->
<body>
  <div id="app">...</div>
  <script id="__ui:server:storage:id__" type="application/json">{"theme":"dark"}</script>
</body>
```

2.  Trong trình duyệt, sau lần gọi đầu tiên đến `get()` hoặc `has()`, lớp sẽ tự động tìm đoạn mã này và khôi phục tất cả các giá trị.

---

> **Quan trọng:** `ServerStorage` tự động thoát (escape) chuỗi `</script>` trong dữ liệu của bạn, ngăn chặn các cuộc tấn công XSS trong quá trình hydration.
