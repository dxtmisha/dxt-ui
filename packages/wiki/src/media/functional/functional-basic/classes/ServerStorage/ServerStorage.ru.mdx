# Класс ServerStorage

`ServerStorage` — это механизм для управления **изолированным глобальным состоянием** в приложениях с поддержкой SSR (Server-Side Rendering).

Класс позволяет работать с данными так же просто, как со статическими переменными, но гарантирует, что на сервере данные разных пользователей не будут смешиваться.

## Основные возможности

1.  **Изоляция запросов**: На сервере данные привязываются к контексту текущего HTTP-запроса.
2.  **Гидратация**: Возможность автоматически передавать состояние с сервера на клиент через JSON.
3.  **Безопасность**: Автоматическое экранирование XSS при передаче данных.

```typescript
import { ServerStorage } from '@dxtmisha/functional-basic'

// Сохранение данных с флагом гидратации (будут доступны на клиенте)
ServerStorage.set('theme', () => 'dark', true)

// Получение данных
const theme = ServerStorage.get('theme') // 'dark'
```

## Методы

### `init`
```typescript
static init(listener: () => Record<string, any>): typeof ServerStorage
```
Инициализирует хранилище слушателем контекста. На сервере этот слушатель должен возвращать объект, уникальный для каждого запроса.

### `get`
```typescript
static get<T = any>(key: string, defaultValue?: () => T, hydration: boolean = false): T
```
Извлекает значение по ключу.
- `defaultValue`: если ключ не найден, вызовется эта функция, результат сохранится и вернется.
- `hydration`: если `true`, значение будет включено в JSON для передачи на клиент.

### `set`
```typescript
static set<T = any>(key: string, value: () => T, hydration: boolean = false): T
```
Записывает значение в хранилище.

### `has`
```typescript
static has(key: string): boolean
```
Проверяет наличие ключа в текущем контексте.

### `toString`
```typescript
static toString(): string
```
Генерирует HTML-тег `<script type="application/json">` со всеми данными, помеченными для гидратации. Этот метод вызывается на сервере в шаблоне страницы.

## Изоляция в SSR (Vue / Nuxt)

Для корректной работы на сервере необходимо передать контекст запроса. В Vue 3.5+ это можно сделать через `useId` или через специальный объект контекста.

```typescript
import { useId } from 'vue'
import { ServerStorage } from '@dxtmisha/functional-basic'

// Инициализация (например, в плагине или в корневом компоненте)
ServerStorage.init(() => {
  // На сервере возвращаем объект, привязанный к запросу
  // На клиенте это может быть просто глобальный объект
  return typeof window === 'undefined' ? (globalContext || {}) : window
})
```

## Гидратация (Передача данных клиенту)

Чтобы данные, установленные на сервере, "ожили" в браузере:

1.  На сервере вставьте результат `ServerStorage.toString()` в HTML (обычно в конец `<body>`).
```html
<!-- Server output -->
<body>
  <div id="app">...</div>
  <script id="__ui:server:storage:id__" type="application/json">{"theme":"dark"}</script>
</body>
```

2.  В браузере при первом вызове `get()` или `has()` класс автоматически найдет этот скрипт и восстановит все значения.

---

> **Важно:** `ServerStorage` автоматически экранирует последовательность `</script>` внутри ваших данных, предотвращая XSS-атаки при гидратации.
