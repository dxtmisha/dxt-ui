# Класс ApiHydration

Класс для сбора данных API для последующей гидратации на стороне клиента при серверном рендеринге (SSR). Он позволяет серверу захватывать ответы API во время начального рендеринга и передавать их клиенту, предотвращая избыточные сетевые запросы при инициализации приложения в браузере.

## Ключевые особенности

- **Оптимизация SSR** — уменьшает количество запросов после загрузки страницы за счет повторного использования серверных данных.
- **Захват данных** — автоматически собирает успешные ответы API, помеченные для глобального использования.
- **Встраивание в HTML** — генерирует безопасный тег `<script>` с данными в формате JSON для вставки в документ.
- **Синхронизация с клиентом** — позволяет `ApiResponse` автоматически загружать гидратированные данные на стороне клиента.

## Инициализация

Для инициализации объекта вызовите конструктор `ApiHydration()`.

```typescript
import { ApiHydration } from '@dxtmisha/functional'

// 1. Инициализация экземпляра
const apiHydration = new ApiHydration()

// 2. Захват ответа (обычно обрабатывается ApiInstance внутренне)
apiHydration.toClient({
  path: 'user/profile',
  method: 'GET',
  global: true
}, { id: 1, name: 'Admin' })
```

## Методы

- `initResponse(response: ApiResponse): void` — Инициализирует экземпляр `ApiResponse` данными гидратации, если выполнение происходит в среде DOM.
- `toClient<T>(apiFetch: ApiFetch, response: T): void` — Сохраняет ответ API для гидратации на стороне клиента. Работает только на сервере для глобальных запросов.
- `toString(): string` — Возвращает строковое представление данных гидратации (в виде тега `<script>`) для включения в HTML-документ.

## Руководство по настройке SSR

Для корректной реализации гидратации выполните следующие шаги для синхронизации данных сервера и клиента.

### 1. Выполнение на стороне сервера (SSR)
Во время рендеринга страницы на сервере все API-запросы, выполненные через `ApiInstance` (с параметром `global: true`), автоматически фиксируются менеджером `ApiHydration`.

### 2. Вставка данных в HTML
Перед отправкой ответа клиенту необходимо вставить захваченные данные в HTML-шаблон, используя метод `getHydrationScript()`.

```typescript
// Пример для SSR-фреймворка или шаблонизатора
const html = `
  <!DOCTYPE html>
  <html>
    <head>
      <title>Мое приложение</title>
      ${Api.getHydrationScript()}
    </head>
    <body>
      <div id="app">${renderedApp}</div>
    </body>
  </html>
`;
```

### 3. Гидратация на стороне клиента
При инициализации приложения на клиенте конструктор `ApiInstance` автоматически обнаруживает тег скрипта и заполняет кэш моков `ApiResponse`. Последующие вызовы тех же самых эндпоинтов API будут мгновенно возвращать гидратированные данные без выполнения реального сетевого запроса.
