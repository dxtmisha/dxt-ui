# Lớp ApiHydration

Một lớp dùng để thu thập dữ liệu API phục vụ cho việc đồng bộ (hydration) ở phía client trong quá trình SSR (Server-Side Rendering). Nó cho phép máy chủ chụp lại các phản hồi API trong lần kết xuất đầu tiên và chuyển chúng tới client, giúp ngăn chặn các yêu cầu mạng dư thừa khi ứng dụng khởi tạo trong trình duyệt.

## Các tính năng chính

- **Tối ưu hóa SSR** — giảm số lượng yêu cầu sau khi tải trang bằng cách tái sử dụng dữ liệu từ phía máy chủ.
- **Thu thập dữ liệu** — tự động thu thập các phản hồi API thành công được đánh dấu để sử dụng toàn cục.
- **Nhúng HTML** — tạo thẻ `<script>` an toàn với dữ liệu được định dạng JSON để nhúng vào tài liệu.
- **Đồng bộ hóa Client** — cho phép `ApiResponse` tự động tải dữ liệu đã đồng bộ ở phía client.

## Khởi tạo

Để khởi tạo đối tượng, hãy gọi hàm khởi tạo `ApiHydration()`.

```typescript
import { ApiHydration } from '@dxtmisha/functional'

// 1. Khởi tạo thực thể
const apiHydration = new ApiHydration()

// 2. Thu thập phản hồi (thường do ApiInstance xử lý nội bộ)
apiHydration.toClient({
  path: 'user/profile',
  method: 'GET',
  global: true
}, { id: 1, name: 'Admin' })
```

## Các phương thức

- `initResponse(response: ApiResponse): void` — Khởi tạo thực thể `ApiResponse` với dữ liệu đồng bộ nếu đang chạy trong môi trường DOM.
- `toClient<T>(apiFetch: ApiFetch, response: T): void` — Lưu phản hồi API để đồng bộ phía client. Chỉ hoạt động trên máy chủ cho các yêu cầu toàn cục.
- `toString(): string` — Trả về chuỗi đại diện của dữ liệu đồng bộ (dưới dạng thẻ `<script>`) để đưa vào tài liệu HTML.

## Hướng dẫn thiết lập SSR

Để thực hiện đồng bộ (hydration) một cách chính xác, hãy làm theo các bước sau để kết nối dữ liệu giữa máy chủ và trình duyệt.

### 1. Thực thi phía Máy chủ (SSR)
Trong khi kết xuất trang trên máy chủ, tất cả các yêu cầu API được thực hiện thông qua `ApiInstance` (với thuộc tính `global: true`) sẽ tự động được trình quản lý `ApiHydration` ghi lại.

### 2. Nhúng dữ liệu vào HTML
Trước khi gửi phản hồi tới client, bạn phải nhúng dữ liệu đã thu thập vào mẫu HTML của mình bằng cách sử dụng `getHydrationScript()`.

```typescript
// Ví dụ cho một SSR framework hoặc template engine
const html = `
  <!DOCTYPE html>
  <html>
    <head>
      <title>Ứng dụng của tôi</title>
      ${Api.getHydrationScript()}
    </head>
    <body>
      <div id="app">${renderedApp}</div>
    </body>
  </html>
`;
```

### 3. Đồng bộ hóa phía Client
Khi ứng dụng khởi tạo trên client, hàm khởi tạo của `ApiInstance` sẽ tự động phát hiện thẻ script và điền vào bộ nhớ cache giả lập của `ApiResponse`. Các lệnh gọi tiếp theo tới cùng một endpoint API sẽ trả về dữ liệu đã đồng bộ ngay lập tức mà không cần thực hiện yêu cầu mạng thực tế.
