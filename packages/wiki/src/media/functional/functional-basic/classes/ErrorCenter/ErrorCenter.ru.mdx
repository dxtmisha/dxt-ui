# ErrorCenter

Статическая обертка для `ErrorCenterInstance`, предоставляющая централизованное управление хранением, извлечением и обработкой ошибок. Он выступает в роли глобального реестра причин ошибок и предоставляет механизм для регистрации и вызова пользовательских обработчиков для различных групп ошибок.

Класс следует паттерну "Статического моста", делегируя все операции синглтон-экземпляру, сохраняя при этом чистый, глобально доступный API.

## Ключевые особенности

- **Статический доступ** — глобальный статус к управлению ошибками без необходимости ручного создания экземпляров.
- **Централизованный реестр** — хранит стандартизированные причины ошибок для единообразной обратной связи.
- **Групповая обработка** — позволяет регистрировать несколько функций обратного вызова для конкретных категорий.
- **Предустановленные значения** — по умолчанию настроен стандартными сетевыми ошибками (`timeout`, `offline`, `notFound` и т.д.).

## Методы действий

### `on`

Запускает обработку указанной ошибки. Метод ищет предварительно зарегистрированные данные для указанного кода ошибки, объединяет их с входной причиной (`cause`), выполняет все зарегистрированные обработчики для этой группы и выводит ошибку в консоль.

**Параметры:**
- `cause: ErrorCenterCauseItem` — Причина для запуска. Код и группа обязательны, если причина была зарегистрирована ранее через `add()`.

**Возвращает:** `void`

## Методы проверки

- `has(code: string, group?: string): boolean` — Проверяет наличие причины ошибки в реестре.
- `get(code: string, group?: string): ErrorCenterCauseItem | undefined` — Извлекает полный объект причины ошибки.

## Конфигурация

- `add(cause: ErrorCenterCauseItem): void` — Добавляет одну причину ошибки в хранилище.
- `addList(causes: ErrorCenterCauseList): void` — Добавляет список причин ошибок в реестр.
- `addHandler(group: ErrorCenterGroup, handler: ErrorCenterHandlerCallback): void` — Регистрирует функцию обратного вызова для группы.
- `addHandlerList(handlers: ErrorCenterHandlerList): void` — Регистрирует набор обработчиков.

## Получение инстанса

- `getItem(): ErrorCenterInstance` — Возвращает базовый экземпляр, используемый оберткой.

## Типы

#### `ErrorCenterGroup`
Идентификатор группы ошибок.
- `string | undefined` — строковое имя группы или `undefined` для глобального обработчика по умолчанию.

#### `ErrorCenterCauseItem`
Детальная информация о причине ошибки.
- `group?: ErrorCenterGroup` — Группа, к которой относится эта ошибка.
- `code: string` — Уникальный код ошибки (например, '404', 'AUTH_ERROR').
- `priority?: number` — Уровень приоритета, используемый для объединения нескольких ошибок (по умолчанию: `500`).
- `label?: string` — Краткое, понятное человеку название ошибки.
- `message?: string` — Описательное сообщение об ошибке.
- `details?: any` — Любой дополнительный контекст или исходные данные ошибки.

#### `ErrorCenterCauseList`
Массив объектов `ErrorCenterCauseItem`.

#### `ErrorCenterHandlerCallback`
Сигнатура функции для обратных вызовов обработки ошибок.
- `(cause: ErrorCenterCauseItem) => void` — получает объект причины ошибки при вызове.

#### `ErrorCenterHandlerItem`
Внутренний объект хранения, связывающий группу с её обработчиками.
- `group?: ErrorCenterGroup` — Целевая группа ошибок.
- `handlers: ErrorCenterHandlerCallback[]` — Список функций обратного вызова для этой группы.

#### `ErrorCenterHandlerList`
Массив объектов `ErrorCenterHandlerItem`, используемый для пакетной регистрации.

## Стандартные причины ошибок

`ErrorCenter` уже содержит стандартные причины ошибок для различных функциональных областей. Они определены в `@dxtmisha/functional-basic/media/errorCauseList`.

### API (`group: 'api'`)
- **`unknown`** — **Неизвестная ошибка**: Произошла непредвиденная ошибка. Пожалуйста, попробуйте позже.
- **`timeout`** — **Тайм-аут запроса**: Запрос занял слишком много времени. Пожалуйста, проверьте соединение.
- **`offline`** — **Нет соединения**: Вы находитесь в автономном режиме. Пожалуйста, проверьте интернет-соединение.
- **`notFound`** — **Не найдено**: Запрошенный ресурс не найден на сервере.
- **`server`** — **Ошибка сервера**: Что-то пошло не так на стороне сервера.
- **`cacheClear`** — **Ошибка очистки кэша**: Произошла ошибка во время фоновой очистки кэша.

### Интернационализация (`group: 'intl'`)
- **`datetime`** — **Ошибка Intl Datetime**: Сбой в логике форматирования даты или времени.
- **`display`** — **Ошибка Intl Display**: Сбой при форматировании отображаемых имен или регионов.
- **`number`** — **Ошибка Intl Number**: Сбой при форматировании чисел или валют.
- **`plural`** — **Ошибка Intl Plural**: Сбой в логике правил множественного числа.
- **`relative`** — **Ошибка Intl Relative**: Сбой при форматировании относительного времени.

### Хранилище (`group: 'storage'`)
- **`context`** — **Ошибка отсутствия контекста**: Контекст хранилища отсутствует (вероятно, не был вызван `init()`). Изоляция данных для этого запроса не удалась.
- **`error`** — **Ошибка хранилища**: Общий сбой при чтении или записи в хранилище.

### Другие группы
- **Broadcast** (`group: 'broadcast'`)
    - **`error`**: Непредвиденная ошибка в обмене данными через `BroadcastChannel`.
- **Hydration** (`group: 'hydration'`)
    - **`error`**: Сбой во время гидратации данных на стороне клиента.
- **Share** (`group: 'share'`)
    - **`error`**: Непредвиденный сбой во время процесса обмена (sharing).
- **Transformation** (`group: 'transformation'`)
    - **`error`**: Сбой во время преобразования или форматирования данных.

## Пример использования

```javascript
import { ErrorCenter } from '@dxtmisha/functional-basic'

// 1. Настройка глобальных обработчиков
ErrorCenter.addHandler('api', (cause) => {
  notification.error({
    title: cause.label,
    message: cause.message
  })
})

// 2. Регистрация специфических причин ошибок
ErrorCenter.add({
  group: 'auth',
  code: 'token_expired',
  label: 'Сессия истекла',
  message: 'Ваша сессия истекла. Пожалуйста, войдите снова.'
})

// 3. Вызов ошибки через реестр
ErrorCenter.on({ group: 'auth', code: 'token_expired' })
```
