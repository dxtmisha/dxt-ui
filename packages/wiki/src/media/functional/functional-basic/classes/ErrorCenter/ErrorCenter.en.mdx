# ErrorCenter

A static wrapper for `ErrorCenterInstance` providing centralized management for error storage, retrieval, and event-based handling. It acts as a global registry for error causes and provides a mechanism to register and trigger custom handlers for different error groups.

The class follows the "Static Bridge" pattern, delegating all operations to a singleton instance while maintaining a clean, globally accessible API.

## Key Features

- **Static Access** — Global access to error management without the need for manual instantiation.
- **Centralized Registry** — Stores standardized error causes (labels and messages) for consistent UI feedback.
- **Group-Based Handling** — Allows registering multiple callbacks for specific error categories.
- **Predefined Defaults** — Comes pre-configured with standard network error causes (`timeout`, `offline`, `notFound`, etc.).

## Action Methods

### `on`

Triggers error handling for the specified cause. It merges the provided cause with stored data (if any), executes all registered handlers for the group, and logs the error to the console.

**Parameters:**
- `cause: ErrorCenterCauseItem` — The cause to trigger. Only `code` and `group` are required if the cause was previously registered via `add()`.

**Returns:** `void`

## Check Methods

- `has(code: string, group?: string): boolean` — Checks if an error cause exists in the registry.
- `get(code: string, group?: string): ErrorCenterCauseItem | undefined` — Retrieves the full error cause object.

## Configuration

- `add(cause: ErrorCenterCauseItem): void` — Adds a single error cause to the global storage.
- `addList(causes: ErrorCenterCauseList): void` — Adds multiple error causes to the registry at once.
- `addHandler(group: ErrorCenterGroup, handler: ErrorCenterHandlerCallback): void` — Registers a callback function for a specific group.
- `addHandlerList(handlers: ErrorCenterHandlerList): void` — Registers a batch of handlers.

## Instance Retrieval

- `getItem(): ErrorCenterInstance` — Returns the underlying instance used by the static wrapper.

## Types

#### `ErrorCenterGroup`
Represents an error group identifier.
- `string | undefined` — A string name for the group, or `undefined` for a global/default handler.

#### `ErrorCenterCauseItem`
Detailed information about an error cause.
- `group?: ErrorCenterGroup` — The group this error belongs to.
- `code: string` — A unique error code (e.g., '404', 'AUTH_ERROR').
- `priority?: number` — Importance level used for merging multiple errors (default: `500`).
- `label?: string` — A short, human-readable name for the error.
- `message?: string` — A descriptive error message.
- `details?: any` — Any additional context or raw error data.

#### `ErrorCenterCauseList`
An array of `ErrorCenterCauseItem` objects.

#### `ErrorCenterHandlerCallback`
The function signature for error handling callbacks.
- `(cause: ErrorCenterCauseItem) => void` — Receives the error cause object when triggered.

#### `ErrorCenterHandlerItem`
An internal storage object mapping a group to its handlers.
- `group?: ErrorCenterGroup` — The targeted error group.
- `handlers: ErrorCenterHandlerCallback[]` — List of callback functions for this group.

#### `ErrorCenterHandlerList`
An array of `ErrorCenterHandlerItem` objects used for batch registration.

## Default Error Causes

`ErrorCenter` is pre-populated with common error causes across different functional areas. These are defined in `@dxtmisha/functional-basic/media/errorCauseList`.

### API (`group: 'api'`)
- **`unknown`** — **Unknown Error**: An unexpected error occurred. Please try again later.
- **`timeout`** — **Request Timeout**: The request took too long. Please check your connection.
- **`offline`** — **No Connection**: You are offline. Please check your network.
- **`notFound`** — **Resource Not Found**: The requested resource could not be found on the server.
- **`server`** — **Internal Server Error**: Something went wrong on the server.
- **`cacheClear`** — **Cache Clearing Error**: An error occurred during background cache cleaning.

### Internationalization (`group: 'intl'`)
- **`datetime`** — **Intl Datetime Error**: Failure in date or time formatting logic.
- **`display`** — **Intl Display Error**: Failure in formatting display names or regions.
- **`number`** — **Intl Number Error**: Failure in number or currency formatting.
- **`plural`** — **Intl Plural Error**: Failure in pluralization rules logic.
- **`relative`** — **Intl Relative Error**: Failure in relative time formatting.

### Storage (`group: 'storage'`)
- **`context`** — **Context Missing Error**: Storage context is missing (likely `init()` not called). Data isolation for this request failed.
- **`error`** — **Storage Error**: General failure when reading from or writing to storage.

### Other Groups
- **Broadcast** (`group: 'broadcast'`)
    - **`error`**: Unexpected error in `BroadcastChannel` communication.
- **Hydration** (`group: 'hydration'`)
    - **`error`**: Failure during client-side hydration.
- **Share** (`group: 'share'`)
    - **`error`**: Unexpected failure during the sharing process.
- **Transformation** (`group: 'transformation'`)
    - **`error`**: Failure during data transformation or formatting.

## Usage Example

```javascript
import { ErrorCenter } from '@dxtmisha/functional-basic'

// 1. Configure global handlers
ErrorCenter.addHandler('api', (cause) => {
  notification.error({
    title: cause.label,
    message: cause.message
  })
})

// 2. Register specific error causes
ErrorCenter.add({
  group: 'auth',
  code: 'token_expired',
  label: 'Session Expired',
  message: 'Your session has timed out. Please log in again.'
})

// 3. Trigger errors via code and group
ErrorCenter.on({ group: 'auth', code: 'token_expired' })
```
