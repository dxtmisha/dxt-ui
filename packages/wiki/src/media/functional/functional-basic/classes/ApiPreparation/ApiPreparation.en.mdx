# ApiPreparation Class

A class specifically designed to handle tasks and side effects immediately running before and after an API request execution. It serves to intercept requests globally via tracking hooks (`callback` and `callbackEnd`), perform state manipulations, or trigger global logic like loading indicators.

## Key Features

- **Pre-request Execution** — hook into the API lifecycle immediately before the actual `fetch` call is made.
- **Post-request Analysis** — execute logic exactly after a request completes, with proper access to the `Response` object and the original request configuration.
- **Concurrency Control** — ensures overlapping hooks are resolved collectively without collision.
- **Data Injection** — post-request hooks optionally return data or a `reset` flag that can manipulate downstream handlers.

## Initialization

Creates a new `ApiPreparation` instance. The class is designed to handle tasks and side effects immediately before and after an API request execution. It allows intercepting requests, manipulating state, or triggering global logic.

```typescript
import { ApiPreparation } from '@dxtmisha/functional'

// 1. Simple initialization
const preparation = new ApiPreparation()

// 2. Setting pre-request callback
preparation.set(async (apiFetch) => {
  console.log('Preparing request to:', apiFetch.path)
})

// 3. Setting post-request callback
preparation.setEnd(async (query, apiFetch) => {
  console.log('Request finished with status:', query.status)
  return { reset: false }
})
```

## Methods

- `make(active: boolean, apiFetch: ApiFetch): Promise<void>` — Executes the preparation callback before executing the request.
- `makeEnd(active: boolean, query: Response, apiFetch: ApiFetch): Promise<ApiPreparationEnd>` — Executes analysis logically after the request finishes.
- `set(callback: (apiFetch: ApiFetch) => Promise<void>): this` — Connects a callback function triggered right before a request execution.
- `setEnd(callback: (query: Response, apiFetch: ApiFetch) => Promise<ApiPreparationEnd>): this` — Connects a callback function executed after request termination.
