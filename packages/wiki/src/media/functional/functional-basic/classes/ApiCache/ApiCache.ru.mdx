# Класс ApiCache

Класс для кэширования ответов API. Он позволяет сохранять результаты запросов в памяти или во внешнем хранилище, предотвращая повторные сетевые вызовы в течение заданного времени. В текущей реализации кэширование активно только на стороне сервера (SSR), чтобы ускорить рендеринг и снизить нагрузку на API при параллельных запросах.

## Ключевые особенности

- **Автоматическое управление ключами** — генерирует уникальные ключи кэша на основе параметров запроса (path, method, request и др.).
- **Гибкое время жизни (TTL)** — позволяет задавать время хранения для каждого запроса индивидуально или использовать значение по умолчанию (24 часа).
- **SSR Оптимизация** — предотвращает лишние вызовы API в Node.js окружении. По умолчанию отключен в браузере.
- **Поддержка слушателей** — возможность подключить внешнее хранилище (например, `ServerStorage` или базу данных) через callback-функции.
- **Фоновая очистка** — автоматически удаляет устаревшие записи при обращении к кэшу.

## Инициализация

Обычно экземпляр `ApiCache` создается и управляется внутри `ApiInstance`, но его можно использовать и самостоятельно.

```typescript
import { ApiCache } from '@dxtmisha/functional-basic'

// 1. Создание экземпляра
const apiCache = new ApiCache()

// 2. Опциональная привязка к внешнему хранилищу
apiCache.init(
  (key) => myStorage.get(key),
  (key, value) => myStorage.set(key, value),
  (key) => myStorage.remove(key)
)
```

### Работа с внешним хранилищем (Пример с Nitro)

По умолчанию `ApiCache` хранит данные в оперативной памяти процесса. Это быстро, но имеет ряд ограничений:
1. **Потеря данных** при перезагрузке процесса сервера.
2. **Отсутствие синхронизации** — в кластерной среде у каждого экземпляра сервера будет свой независимый кэш.
3. **Холодные старты** — в serverless-окружениях (Vercel, AWS Lambda) память может очищаться между вызовами.

Именно для этого нужен метод `init`. Он позволяет «подложить» под капот `ApiCache` любое внешнее хранилище (Redis, файловую систему, базу данных), которое поддерживает асинхронную работу.

Ниже приведен пример использования `ApiCache` совместно с Nitro (движком Nuxt) через встроенный механизм `useStorage`:

```typescript
// server/plugins/api-cache.ts
export default defineNitroPlugin(() => {
  const cache = Api.getCache()
  const storage = useStorage('cache') // Регион кэша в Nitro

  cache.init(
    async (key) => await storage.getItem(key),
    async (key, value) => await storage.setItem(key, value),
    async (key) => await storage.removeItem(key)
  )
})
```

Благодаря тому, что методы `ApiCache` теперь асинхронны, они корректно дожидаются выполнения операций от внешних систем.

### Параметры init

Метод `init` принимает три колбэка для интеграции с внешним хранилищем:
- `getListener: (key: string) => any` — функция для получения данных. Должна возвращать объект записи кэша или `undefined`.
- `setListener: (key: string, value: ApiCacheItem) => void` — функция для сохранения данных. Вызывается при каждой записи в кэш.
- `removeListener: (key: string) => void` — функция для удаления данных. Вызывается при истечении срока жизни или принудительном удалении.

## Методы

### Основные методы
- `get<T>(key: string): T | undefined` — Получение данных по ключу.
- `getByFetch<T>(fetch: ApiFetch): T | undefined` — Получение данных на основе параметров запроса API.
- `set<T>(key: string, value: T, age?: number): this` — Сохранение данных в кэш.
- `setByFetch<T>(fetch: ApiFetch, value: T): this` — Сохранение данных на основе параметров запроса.
- `remove(key: string): this` — Удаление записи из кэша.

### Интеграция с ApiInstance
При использовании через `ApiInstance` достаточно указать параметр `cache` в секундах:

```typescript
// Данные будут закэшированы на 60 секунд на стороне сервера
const data = await Api.get({
  path: 'v1/products',
  cache: 60
})
```

## Как это работает (SSR)

1. **Первый запрос**: `ApiInstance` проверяет наличие данных в `ApiCache`. Если данных нет, выполняется реальный сетевой запрос.
2. **Сохранение**: После успешного ответа данные сохраняются в `ApiCache` с указанным временем жизни.
3. **Повторный запрос**: Если в течение TTL выполняется такой же запрос, `ApiInstance` мгновенно возвращает данные из кэша, не дожидаясь сетевого ответа.
4. **Валидация**: `ApiCache` постоянно проверяет актуальность данных (`cacheAge` + `age`). Если время истекло, запись считается недействительной и будет удалена.
