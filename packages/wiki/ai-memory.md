# Правила оформления документации для классов (.mdx)

При написании документации для классов (например, `Api`, `ApiCache`, `ApiStatus`), строго соблюдайте разделение на короткую и длинную версии:

**1. Короткая версия (Справочник методов)**
Обязательно размещается в едином разделе `## Методы`.
- Методы логически группируются под заголовками третьего уровня (например, `### Действие`, `### Конфигурация`, `### Аксессоры`).
- Каждый метод описывается маркированным списком в формате:
  `- method<T>(arg: Type): ReturnType — Краткое описание.` (сигнатура выделяется инлайн-кодом).

**2. Длинная версия (Подробное руководство)**
- Размещается в виде самостоятельных разделов второго уровня (например, `## Инициализация и внешнее хранилище`, `## Архитектура`).
- Содержит подробное текстовое объяснение логики работы.
- Важные концепции и предупреждения выделяются цитатой: `> Важно: ...`
- Обязательно сопровождается блоками примеров кода: ` ```javascript ... ``` `.
